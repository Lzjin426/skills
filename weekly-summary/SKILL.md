# Weekly Summary

## 适用范围

- 默认假设当前工作区根目录下存在 `05-note/<year>/Daily/*.md` 与 `05-note/<year>/Weekly/*.md`。
- 若用户明确给出其他文档库根目录，按用户指定路径执行。
- 若本地不存在 `05-note`，先告知缺少数据入口，再让用户确认正确路径。

## 工作流

1. 解析周区间
- 如果用户明确给出区间，优先接受以下两类格式：
  - 旧标题格式：`1.19～1.25-26`
  - ISO 文件名格式：`2026-01-19～2026-01-25`
- 如果用户说“本周/这周”，按周一到周日计算。
- 如果用户说“上周”，使用上一周周一到周日。
- 优先使用脚本生成标准周窗口：

```bash
python3 scripts/week_window.py --which current
python3 scripts/week_window.py --which last
python3 scripts/week_window.py --which explicit --start 2026-01-19 --end 2026-01-25
python3 scripts/week_window.py --which title --title '1.19～1.25-26'
```

2. 扫描本地日报与周报
- 用 `scripts/collect_local_notes.py` 扫描 `05-note`，一次性拿到：
  - 本周应纳入的日报路径
  - 缺失日期
  - 建议输出的周报路径
  - 最近 2~3 篇历史周报路径，作为文风参考
- 示例：

```bash
python3 scripts/collect_local_notes.py --root . --start 2026-01-19 --end 2026-01-25
```

3. 读取并提炼日报
- 逐篇读取已命中的日报 Markdown。
- 重点提炼：
  - 本周完成事项
  - 阶段性成果
  - 问题与风险
  - 下周计划线索
- 若个别日期无日报，只记录为缺失，不补写、不猜测。

4. 抽取本地周报风格
- 读取最近 2~3 篇历史周报作为风格基准，优先参考当前年份 `Weekly` 目录中的近邻文件。
- 只借鉴句式、标题层级、篇幅密度，不继承旧事实。
- 默认保持本地周报格式：
  - 文件名使用 ISO 区间：`YYYY-MM-DD～YYYY-MM-DD.md`
  - 正文标题使用旧标题格式：`# M.DD～M.DD-YY`
  - 保留本地 frontmatter：

```md
---
tags:
aliases:
---
```

5. 生成周总结内容
- 以 `references/weekly-summary-template.md` 为结构骨架，不机械套用。
- 严格基于原始日报，不杜撰结论。
- 术语尽量沿用原文，保持项目/论文语境一致。
- 默认输出 4 个一级章节：
  - `## 主要完成工作`
  - `## 关键阶段性成果`
  - `## 存在问题与风险`
  - `## 下周工作计划`
- `主要完成工作`下的二级小节数量按当周内容动态决定，禁止硬凑。
- 当周主线为论文推进时，论文相关内容默认合并为一个二级小节，例如“论文工作推进”。
- 默认文风为任务导向、低主语，不连续使用第一人称“我”。
- 若用户明确要求“可追溯明细”，可追加“每日工作映射”；否则默认不写。
- 若存在缺失日期，可在结尾用一句话简短说明“某些日期无日报，未纳入总结”，不要扩展推断。

6. 写回本地周报
- 目标路径使用脚本给出的 `weekly_output_path`。
- 若同名文件已存在：
  - 默认先读取旧文件判断是否为已完成周报。
  - 未经用户明确允许，不直接覆盖。
  - 优先改写为 `_v2.md` 或在答复中请用户确认覆盖策略。
- 写入内容前确保父目录存在。

7. 结果回传给用户
- 返回生成的周报路径。
- 列出纳入的日报日期。
- 列出缺失日期（若有）。
- 若发生覆盖规避，明确说明最终写入的文件名。

## 执行细节

1. 本地日报定位规则
- 日报路径固定为：`05-note/<year>/Daily/YYYY-MM-DD.md`
- 周报路径固定为：`05-note/<year>/Weekly/YYYY-MM-DD～YYYY-MM-DD.md`

2. 时间规则
- 默认周一为起始日，周日为结束日。
- 检索与写入必须保持在同一自然周内，避免跨周混入。

3. 兼容旧命名
- 用户若提到 `1.19～1.25-26` 这类旧周标题，只作为输入解析格式。
- 新生成的本地文件名一律使用 ISO 区间格式，正文标题仍使用旧标题格式以保持阅读连续性。

4. 缺失数据处理
- 缺日报时仅标记缺失，不从其他来源补齐。
- 不得根据相邻日期内容推断缺失日发生的事项。

## 资源

- `scripts/week_window.py`: 解析周区间，输出标准日期范围、正文标题、文件名。
- `scripts/collect_local_notes.py`: 扫描本地 `05-note`，返回日报命中情况、建议输出路径、历史周报参考路径。
- `references/weekly-summary-template.md`: 本地周报模板骨架。
