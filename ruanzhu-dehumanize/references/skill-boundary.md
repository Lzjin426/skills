# Skill 边界说明

本文件说明 `ruanzhu-dehumanize` 与其他相关 skill 的调用边界，避免混淆。

## 与 `humanizer-zh` 的区分

| 场景 | 用 `ruanzhu-dehumanize` | 用 `humanizer-zh` |
|------|------------------------|-------------------|
| Python 代码去 AI 化（软著源代码文档） | 是 | 否 |
| 中文技术文档去 AI 化（用户手册、设计说明书） | 是 | 否 |
| 通用中文散文、小说、营销文案去 AI 化 | 否 | 是 |
| 软著材料中的非技术文本（如申请说明信） | 否 | 是 |
| 需要清理版权红线、作者信息、日期等 | 是 | 否 |
| 需要保持程序-文档对应性 | 是 | 否 |

## 核心区别

- **`humanizer-zh`**：通用文本去 AI 味，侧重语气、句式、修辞的自然化。不处理代码，不管软著规则。
- **`ruanzhu-dehumanize`**：软著申请专用，在通用去 AI 味的基础上，额外处理：
  - 版权红线（版权声明、作者、日期、License 等）
  - 代码特定的 AI 痕迹（docstring、类型注解、match/case 等）
  - 程序-文档对应性校验约束
  - 技术文档特有的 AI 口癖和模板化结构

## 调用建议

如果用户明确提到"软著""软件著作权""源代码文档""用户手册"等关键词，**优先使用本 skill**。

如果用户只是泛泛地说"这段文字 AI 味太重帮我改改"，且内容不是技术文档或代码，**用 `humanizer-zh`**。
