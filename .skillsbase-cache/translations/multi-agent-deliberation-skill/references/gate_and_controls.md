# 门控与调控机制（v1.1）

## 1) 激活门控检查清单

仅当以下三个条件全部满足时，方可启动辩论：

1. 存在可论证的替代方案。
2. 存在可检验的评估标准。
3. 存在可证伪的证据链。

若门控评分 < 3/3，则不应启动多智能体辩论。  
返回包含 `STATUS=GATE_FAILED` 的备用方案。

备用模板：

```text
[STATUS]
GATE_FAILED

[GATE_SCORE]
x/3

[MISSING_CONDITIONS]
- ...

[SINGLE_AGENT_ANSWER]
...

[WHAT_EVIDENCE_IS_NEEDED_TO_ACTIVATE_DEBATE]
...
```

## 2) 预算控制

1. 默认辩论轮数：2。
2. 最大辩论轮数：3。
3. 第 0 阶段为必需阶段，但不计入轮数统计。
4. 限制每个角色的响应长度及每项任务的总令牌预算。
5. 若超出预算，则停止并输出当前最佳答案，同时标记明确的风险标识。

## 3) 停止规则

1. 裁判在每轮结束时给出 0-100 的总分。
2. `score_gain = 当前总分 - 前一轮总分`。
3. 若连续两轮 `score_gain` < 1 分，则停止。
4. 若未添加新证据，则停止。
5. 若分歧源于价值观差异（非事实性），则升级至人工决策。

## 4) 故障模式与缓解措施

1. 共同偏见放大：
   - 在第 0 阶段保持 A/B 隔离。
   - 要求进行独立证据核查。
   - 强制设置反例槽位。

2. 角色崩溃：
   - 强制输出角色特定的内容。
   - 防止早期交叉阅读。

3. 自信但错误的共识：
   - 裁判必须依据证据图谱而非一致率进行判断。

4. 辩论循环：
   - 强制实施轮数上限与停止规则。

## 5) 高风险事项升级机制

针对医疗、法律、金融、安全或重大损失场景：

1. 发布前必须进行人工介入审查。
2. 若问题未解决或证据不足，则输出 `STATUS=INSUFFICIENT_EVIDENCE`。
3. 提供具体的验证计划。
