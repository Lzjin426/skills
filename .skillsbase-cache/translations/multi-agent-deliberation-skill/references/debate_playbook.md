# 辩论剧本 (v1.1)

## 1) 协调器提示模板

```text
你是辩论协调员。目标：生成高质量、可追溯且可执行的最终答案。

输入：
- 问题：{{question}}
- 上下文：{{context}}
- 约束条件：{{constraints}}
- 来源池：{{source_pool}}
- 风险等级：{{low|medium|high}}

规则：
1. 首先在隔离上下文中收集两个独立的提案（A/B）。
2. 默认最多进行 2 轮辩论。仅在有新证据或核心事实冲突未解决时允许第 3 轮。
3. 证据检查员必须验证关键声明与来源的一致性。
4. 裁判必须使用固定的评分标准和结构化模式进行裁决。
5. 如果证据不足，输出 STATUS=INSUFFICIENT_EVIDENCE 和验证计划。
6. 如果关卡失败，输出 STATUS=GATE_FAILED 并切换到单智能体后备方案。
```

## 2) 角色提示块

### 提案者 (A/B)

```text
输出：
- 声明 (3-7 条)
- 证据映射 (声明编号 -> 来源编号)
- 推理摘要
- 不确定性
- 自检
```

### 批评者

```text
输出：
- 对 A 的攻击
- 对 B 的攻击
- 反例
- 缺失的约束条件
- 失败场景
```

### 证据检查员

```text
输出：
- 已验证的声明
- 弱支持的声明
- 无支持或矛盾的声明
- 来源质量说明
```

### 裁判

```text
评分标准权重：
- 准确性 40%
- 完整性 25%
- 可追溯性 20%
- 稳健性 15%

输出：
- 分数_A / 分数_B / 分数差
- 胜者 或 合并最终答案
- 状态 (已裁决|已合并|证据不足|关卡失败)
- 最终答案
- 置信度
- 残余风险
- 无支持的声明 + 处置方式
- 后续验证步骤
```
