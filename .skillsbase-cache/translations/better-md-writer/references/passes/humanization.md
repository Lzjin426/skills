# 更好的 Markdown 写作流程：人性化处理

人性化处理是可选的。默认设置为 `关闭`。

启用时，请使用 [../../../humanizer-zh/SKILL.md](../../../humanizer-zh/SKILL.md) 作为清理参考。

## 使用场景

此流程适用于：
- 教程
- 知识性文章
- 解释性文本
- 面向读者的文档，其中机械化的措辞会影响可读性

## 避免使用场景

以下情况请跳过此流程：
- RFC 和规范文档
- 审计或合规性文件
- 会议纪要和跟踪记录
- 引用或引文较多的文本
- 严格遵循模板的文档

## 目标

目标并非让文本显得随意。目标是去除机械化的措辞，同时保留：
- 准确含义
- 范围和限定条件
- 可追溯性
- 用户的原始观点

## 防护原则

如果清理操作削弱了表达的精确性，请恢复原有措辞。
