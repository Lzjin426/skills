# 更好的 Markdown 写作流程：认知路径

认知路径设计是可选的，默认为 `关闭`。

仅当文档需要在较长篇幅的解释中构建可复用的心智模型时，才启用此功能。

## 何时使用

在需要读者理解孤立事实之外的较长解释、教程或分析中使用此流程。

## 何时跳过

以下情况请跳过此流程：
- 笔记和记录
- 配置文档和操作手册
- 短篇文档
- 以浏览为主的参考资料

## 启用时需要添加的内容

### 核心概念

确定一个能让文档其余部分豁然开朗的心智模型，并在草稿中明确表达出来。

### 认知弧线

将草稿构建为：
- **定位**：展示整体领域以及读者将理解的内容
- **构建**：按依赖顺序添加细节
- **收尾**：回到开篇框架，让读者能够验证理解

### 章节间递进

使相邻章节之间产生连接感：
- 每个章节应回答前一章节提出的问题
- 如果章节可以随意调换顺序，则递进关系过于薄弱

## 注意事项

不要仅仅因为主题技术性强就启用此流程。仅在读者确实能从明确的学习路径中受益时才使用它。
