# 更优 Markdown 写作流程：认知路径

认知路径设计是可选的，默认状态为 `关闭`。

仅当文档需要在较长篇幅的解释中构建可复用的心智模型时，才启用此功能。

## 使用场景

在以下情况下使用此流程：
- 长篇解释、教程或分析
- 读者需要的不只是孤立的事实，而是连贯的理解框架

## 跳过场景

在以下情况下跳过此流程：
- 笔记与记录
- 配置文档与操作手册
- 短篇文档
- 以快速查阅为导向的参考资料

## 启用后需添加的内容

### 核心概念锚点

找出那个能让文档其余部分豁然开朗的核心心智模型，并在草稿中明确点出。

### 认知弧线

将草稿结构塑造为：
- **定位**：展示整体领域，并说明读者将理解什么
- **构建**：按依赖顺序逐步添加细节
- **收尾**：回归开篇框架，让读者能验证自己的理解

### 章节间递进

让相邻章节之间产生连接感：
- 每个章节应回答前一章节提出的问题
- 若章节顺序可随意调换，则递进关系过于薄弱

## 注意事项

不要仅仅因为主题技术性强就启用此流程。仅当读者确实能从明确的学习路径中受益时才使用它。
