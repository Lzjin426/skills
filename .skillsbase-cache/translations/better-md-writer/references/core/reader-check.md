# Better Md Writer Core：读者检查

在最终交付前运行此检查。

## 五个必答问题

1. 读者能否快速了解本文档为何存在？
2. 文档结构是否与其目的相符？
3. 是否有任何重要信息被默认但从未明确说明？
4. 是否有任何部分仅为填充结构而无实质信息？
5. 读者是否知道接下来该做什么、决定什么或理解什么？

如果任何答案不够明确，请在完成前修改草稿。

## 可选流程检查

仅运行与已启用流程匹配的检查：

- **研究流程启用**
  - 事实是否已融入文档自身的表达风格，而非直接粘贴原始内容？
  - 研究是否以有意义的方式改变或加强了草稿？

- **认知路径流程启用**
  - 心智模型是否明确且有用？
  - 章节顺序是否确实有助于理解逐步建立？

- **插图流程启用**
  - 视觉内容是否真正减少了解释成本？
  - 文档是否仍能在不过度重复解释的情况下被理解？
  - 是否有独立的插图审查，还是仅在起草过程中临时决定视觉内容？
  - 每张插图是否有明确的作用，而非仅因“感觉有张图不错”而存在？
  - 如果文档触发了强制插图审查的条件，是否有明确的图像计划？

- **知识库文章**
  - 起草前是否明确了文章子类型：概念解释、比较、结果解读，还是方法/流程？
  - 文章是否回答了一个明确的判断性问题，而非仅收集事实？
  - 是否至少有一个实际或计划中的视觉锚点？
  - 读者能否想象出事物的样貌，而不仅仅是重复定义？
  - 技术术语是否在符号、变量名或实现细节出现前已用通俗语言解释？
  - 合并薄弱章节是否会使文章更易于理解？
  - 文章是否保持专业和中立，未陷入口语化的教学提示或口号式标题？

- **人性化流程启用**
  - 清理工作是否减少了机械化的措辞，同时未削弱精确性、范围或可追溯性？

## 最终规则

如果某个流程未改进文档，应移除其影响，而非为流程本身辩护。
