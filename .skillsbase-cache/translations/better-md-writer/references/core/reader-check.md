# Better Md Writer Core：读者检查

在最终交付前运行此检查。

## 五个必答问题

1.  读者能否快速了解本文档存在的目的？
2.  文档结构是否符合其任务需求？
3.  是否有任何重要信息被默认已知但从未明确说明？
4.  是否有任何部分是仅为了填充结构而无实际信息的“水货”？
5.  读者是否知道接下来该做什么、决定什么或理解什么？

如果任何问题的答案不够有力，请在完成前修订草稿。

## 可选流程检查

仅运行与已启用流程匹配的检查：

-   **研究流程启用**
    -   事实是否已融入文档自身的叙述风格，而非生硬粘贴？
    -   研究是否以有意义的方式改变或加强了草稿？

-   **认知路径流程启用**
    -   心智模型是否明确且有用？
    -   章节顺序是否确实有助于理解随时间逐步建立？

-   **插图流程启用**
    -   视觉元素是否真正减少了解释成本？
    -   文档是否在不过度重复解释的情况下仍然易于理解？
    -   是否有独立的插图审查，还是仅在起草过程中临时决定？
    -   每个视觉元素是否有明确的任务，而非仅仅因为“感觉有张图不错”而存在？
    -   如果文档触发了强制性的插图审查要求，是否有明确的图像计划？

-   **知识库文章**
    -   起草前是否明确了文章子类型：概念解释、比较、结果解读，还是方法/流程？
    -   文章是否回答了一个明确的判断性问题，而非仅仅收集事实？
    -   是否至少有一个实际存在或计划中的视觉锚点？
    -   读者能否想象出事物的样子，而不仅仅是复述定义？
    -   技术术语是否在用符号、变量名或实现细节接管之前，已用通俗语言解释清楚？
    -   合并薄弱章节是否会使文章更易于跟进？
    -   文章是否保持专业和中立，没有滑入口语化的教学提示或口号式的标题？

-   **人性化流程启用**
    -   清理工作是否减少了机械化的措辞，同时没有削弱精确性、范围或可追溯性？

## 最终规则

如果某个流程未能改进文档，请移除其影响，而不是为流程本身辩护。
