# 更好的 Markdown 写作本地默认设置：Fullstop Obsidian

仅当输出内容用于 Fullstop 本地知识库或 Obsidian 工作流时，才应用此文件。

## 知识库默认设置

- 如果用户未指定其他位置，文档、HTML 文件及类似输出应存储在 `Desktop/Fullstop_base` 中。
- 每日和每周笔记默认存储在 `Fullstop_base/03-note`。
- 图片和附件默认存储在 `Fullstop_base/Attachments`。

## Obsidian 工作流

- 当输出内容属于知识库时，默认使用 Obsidian 作为阅读器和验证界面。
- 在适合任务的情况下，优先使用 `obsidian` 命令行工具作为打开或查看知识库内容的实用方式。
- Obsidian 特定行为是本地默认设置，而非全局写作规则。

## 本地 Markdown 偏好设置

以下是 Fullstop 的默认设置，并非适用于所有文档的硬性规则：
- 对于写入 Fullstop 知识库的文档，默认优先使用显式的层级编号标题：
  - 一级标题：`# 1. 标题`
  - 二级标题：`## 1.1 标题`
  - 三级标题：`### 1.1.1 标题`
- 保持编号连续且结构正确；若无明确理由，同一层级不应混用编号与非编号标题
- 面向概念的知识库文章也应使用编号语义标题，除非目标文档已有冲突的内部风格
- 在较长的知识库笔记中，可使用顶级 `---` 分隔符以提高可读性
- 图片应使用稳定、易读的名称
- 前置元数据应存储结构性元数据，而非大量链接声明
- 不要使用前置元数据中的维基链接（如 `related: [[...]]`）或冗长的 `related` 列表来创建图谱边
- 如需关系元数据，优先使用纯文本结构字段，如 `note_type`、`layer`、`prerequisites` 和 `next`
- 将实际的维基链接保留在正文内容中，且仅在它们为读者表达有意义的语义关系时才添加

## Excalidraw 嵌入语法

在 Obsidian 中嵌入 Excalidraw 内容时，请使用：

```markdown
![[filename.excalidraw.md|800|center]]
```

使用 `center` 进行居中显示。默认宽度为 `800`，除非用户另有要求。
