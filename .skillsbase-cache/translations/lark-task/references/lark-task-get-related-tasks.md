# task +get-related-tasks

> **前置要求：** 请先阅读 `../lark-shared/SKILL.md` 以了解身份验证、全局参数和安全规则。
>
> **⚠️ 注意：** 此 API 必须使用用户身份调用。**请勿使用应用身份，否则调用将失败。**
>
> **分页 / 时间游标规则：**
> 在 `+get-related-tasks` 中，`page_token` 是任务 `updated_at` 的微秒级时间游标。
>
> **执行优先级：**
> 1. 如果请求包含开始/结束时间边界（例如，“今年以来”、“最近一个月”、“从 3 月 1 日开始”），首先将**开始时间**边界转换为微秒级 `page_token` 并从该游标开始查询。
> 2. 使用返回的 `page_token` 继续分页，直到 `has_more=false`，但总获取页数不得超过 40。
> 3. 对于有时间边界的查询，**不要**默认使用 `--page-all`。
>
> 仅在以下情况下从头开始使用 `--page-all`：
> 1. 用户明确要求扫描所有相关任务，或
> 2. 无法从请求中推断出任何时间边界。

列出与当前用户相关的任务。

## 推荐命令

```bash
# 列出所有相关任务
lark-cli task +get-related-tasks

# 从某个页面令牌开始，列出未完成的相关任务
lark-cli task +get-related-tasks --include-complete=false --page-token "1752730590582902"

# 仅显示由我创建的任务
lark-cli task +get-related-tasks --created-by-me
```

## 参数

| 参数 | 是否必需 | 描述 |
|-----------|----------|-------------|
| `--include-complete=<bool>` | 否 | 默认行为包含已完成的任务。设置为 `false` 以仅保留未完成的任务。 |
| `--page-all` | 否 | 自动遍历所有页面（最多 40 页）。 |
| `--page-limit <int>` | 否 | 最大页面限制（默认 20）。 |
| `--page-token <string>` | 否 | 从指定的页面令牌开始。此令牌是任务最后更新时间的微秒级游标。 |
| `--created-by-me` | 否 | 仅保留创建者为当前用户的任务。这是在获取相关任务页面后应用的客户端过滤器。 |
| `--followed-by-me` | 否 | 仅保留当前用户关注的任务。这是在获取相关任务页面后应用的客户端过滤器。 |

> **页面令牌说明：** 在 `+get-related-tasks` 中，`page_token` 是一个微秒级的游标，代表任务的最后更新时间。例如，`1752730590582902` 应被视为更新时间的游标，而不是任务 ID。
>
> **客户端过滤器的分页说明：** 当使用 `--created-by-me` 或 `--followed-by-me` 时，过滤操作会在获取每个上游相关任务页面后在本地进行。返回的 `has_more` 和 `page_token` 仍然描述上游游标，因此后续页面可能包含更多匹配的任务，也可能不包含任何匹配项。

## 工作流程

1. 确定用户是需要所有相关任务还是经过过滤的子集。
2. 执行 `lark-cli task +get-related-tasks ...`
3. 报告匹配的任务，如果存在，则同时报告下一个 `page_token`。
