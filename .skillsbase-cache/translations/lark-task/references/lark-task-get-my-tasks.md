# task +get-my-tasks

如果用户查询仅指定了任务名称（例如“完成任务 Lobster No. 1”），请使用此命令按任务摘要列出并搜索任务。

> **前提条件：** 请阅读 `../lark-shared/SKILL.md` 以了解身份验证、全局参数和安全规则。
> 
> **⚠️ 注意：** 此 API 必须使用用户身份调用。**请勿使用应用身份，否则调用将失败。**
>
> **输出渲染说明：**
> 1. 如果需要展示用户字段（负责人、创建者等），不要仅输出原始的 `id`（例如 open_id）。还应尝试解析并显示用户的真实姓名（例如通过联系人技能）以提高可读性。
> 2. 渲染时间戳时（例如创建时间、截止时间），请使用本地时区。格式为 2006-01-02 15:04:05。

列出分配给当前用户的任务，支持按完成状态、创建时间和截止日期进行过滤。
默认情况下，该命令将自动分页最多 20 次。使用 `--page-all` 可获取更多结果（最多 40 页）。

## 推荐命令

```bash
# 按名称搜索特定任务
lark-cli task +get-my-tasks --query "Lobster No. 1"

# 获取我的所有任务（默认获取最多 20 页）
lark-cli task +get-my-tasks

# 获取我未完成的任务（默认获取最多 20 页）
lark-cli task +get-my-tasks --complete=false

# 获取我的所有任务（最多 40 页）
lark-cli task +get-my-tasks --page-all

# 获取最多 10 页
lark-cli task +get-my-tasks --page-limit 10

# 从已知的页面令牌恢复
lark-cli task +get-my-tasks --page-token "pt_xxx"
```

## 参数

| 参数 | 是否必需 | 描述 |
|-----------|----------|-------------|
| `--query <string>` | 否 | 按摘要搜索任务。如果存在完全匹配则返回；否则返回部分匹配。 |
| `--complete=<bool>` | 否 | 可选。如果不提供，则获取所有任务（包括未完成和已完成）。设置为 `true` 仅获取已完成任务，或 `false` 仅获取未完成任务。 |
| `--created_at <string>` | 否 | 查询在此时间之后创建的任务。支持日期格式：`YYYY-MM-DD`、相对格式：`-2d` 或毫秒时间戳。 |
| `--due-start <string>` | 否 | 查询截止时间在此时间之后的任务。支持日期格式：`YYYY-MM-DD`、相对格式：`-2d` 或毫秒时间戳。 |
| `--due-end <string>` | 否 | 查询截止时间在此时间之前的任务。支持日期格式：`YYYY-MM-DD`、相对格式：`-2d` 或毫秒时间戳。 |
| `--page-all` | 否 | 自动遍历所有页面（最多 40 页）。 |
| `--page-limit <int>` | 否 | 最大页面限制（默认 20）。 |
| `--page-token <string>` | 否 | 从指定的页面令牌开始（适用于恢复之前的查询）。 |

## 工作流程

1. 根据用户的请求确定过滤器。
2. 执行命令。CLI 将自动循环至指定限制（默认 20，或使用 `--page-all` 时为 40）以获取记录。
3. 显示结果（ID、摘要、截止时间和创建日期）。
