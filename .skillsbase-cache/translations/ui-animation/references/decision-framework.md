# 动画决策框架

在编写任何动画代码之前，请按顺序回答以下四个问题。

## 1. 这里是否应该使用动画？

**用户会多频繁地看到这个动画？**

| 频率 | 示例 | 决策 |
|---|---|---|
| 每天 100 次以上 | 键盘快捷键、命令面板切换 | **绝不**使用动画。 |
| 每天数十次 | 悬停效果、列表导航 | 移除或大幅简化动画 |
| 偶尔 | 模态框、抽屉、提示条 | 使用标准动画 |
| 罕见 / 首次 | 新手引导、反馈表单、庆祝效果 | 可以增添愉悦感 |

切勿为键盘触发的操作添加动画。这些操作每天重复数百次——动画会让它们感觉缓慢且不连贯。

## 2. 动画的目的是什么？

在编写代码之前，先回答“为什么要添加这个动画？”

| 目的 | 描述 | 示例 |
|---|---|---|
| **反馈** | 确认用户操作已被接收 | 按钮按下时的缩放、切换状态 |
| **导向** | 展示空间关系 | 抽屉从边缘滑出、菜单从触发器处缩放展开 |
| **连续性** | 在状态变化间保持上下文 | 页面过渡、布局变化 |
| **愉悦感** | 增添个性（谨慎使用） | 错开显示、弹簧过冲效果 |

如果目的仅仅是“看起来很酷”，并且用户会频繁看到它，那就不要添加动画。

## 2. 应该使用何种缓动效果？

遵循以下决策树：

- **进入视口？** → 进入曲线：`cubic-bezier(0.22, 1, 0.36, 1)`
- **退出视口？** → 使用相同曲线，但缩短持续时间
- **在屏幕上移动/滑动？** → 移动曲线：`cubic-bezier(0.25, 1, 0.5, 1)`
- **简单悬停（颜色/透明度）？** → `200ms ease`
- **需要物理感？** → 弹簧动画
- **直接操作（拖拽）？** → 无缓动，跟随指针
- **持续运动（跑马灯、旋转器）？** → `linear`

使用自定义缓动曲线。避免在 UI 中使用 `ease-in`。

**缓动资源：** [easing.dev](https://easing.dev/) 和 [easings.co](https://easings.co/) 可获取更强烈的自定义变体。

## 4. 动画应该多快？

| 元素 | 持续时间 |
|---|---|
| 按钮按下反馈 | 100–160ms |
| 工具提示、小型弹出框 | 125–200ms |
| 下拉菜单、选择器 | 150–250ms |
| 模态框、抽屉 | 200–350ms |
| 页面过渡 | 250–400ms |
| 说明性 / 营销性动画 | 最长 1000ms |

**规则：常规 UI 动画保持在 300ms 以内。**

### 感知性能

动画速度会影响感知性能：

- 快速旋转的加载器会让加载感觉更快（实际加载时间相同，但感知不同）
- 200ms 的 `ease-out` 比 200ms 的 `ease-in` **感觉**更快，因为用户能立即看到运动
- 在第一个工具提示打开后，后续工具提示立即显示（跳过延迟和动画）会让工具栏感觉更快

### 非对称时序

进入动画可以比退出动画稍慢。例如：长按删除使用 2s 线性动画按下，200ms 缓出动画释放。

```css
/* 释放：快速 */
.overlay {
  transition: clip-path 200ms ease-out;
}

/* 按下：缓慢且慎重 */
.button:active .overlay {
  transition: clip-path 2s linear;
}
```
