# 弹簧动画

相比基于时长的动画，弹簧动画模拟了真实的物理效果，因此感觉更加自然。它们没有固定的持续时间——其运动状态取决于物理参数。

## 何时使用弹簧动画

- 具有动量的拖拽交互（释放后让物理效果接管）
- 需要感觉“有生命力”的元素（如苹果的动态岛）
- 可在动画中途被打断的手势操作
- 装饰性的鼠标跟踪交互
- 过冲效果（趣味性 UI）

**不要将弹簧动画用于：** 简单的淡入淡出、颜色过渡，或需要精确计时的 UI。

## 弹簧参数

| 参数 | 控制内容 | 典型范围 |
|---|---|---|
| `stiffness` | 运动速度（值越高越快） | 100–500 |
| `damping` | 阻力（值越低弹跳越多） | 15–40 |
| `mass` | 重量感（值越高越慢、越重） | 0.5–2 |

## 配置预设

**苹果风格（推荐——更易于理解）：**

```js
{ type: "spring", duration: 0.5, bounce: 0.2 }
```

**传统物理参数（控制更精细）：**

| 预设 | stiffness | damping | 使用场景 |
|---|---|---|---|
| 迅捷（苹果默认） | 500 | 40 | 通用 UI，无弹跳 |
| 弹跳 | 300 | 20 | 趣味元素、通知 |
| 柔和 | 200 | 30 | 页面过渡、大型元素 |
| 僵硬 | 700 | 50 | 小型精确移动 |

使用弹跳效果时保持微妙（0.1–0.3）。在大多数 UI 场景中应避免弹跳。

## 可中断性优势

弹簧动画在被打断时能保持速度——而 CSS 关键帧动画会从零重新开始。这使得弹簧动画非常适合用户可能在运动中途改变的手势操作。

```tsx
// 弹簧动画能从当前位置平滑反向运动
<motion.div
  animate={{ transform: isOpen ? "translateX(0)" : "translateX(-100%)" }}
  transition={{ type: "spring", stiffness: 500, damping: 40 }}
/>
```

## 基于弹簧的鼠标交互

将数值直接绑定到鼠标位置会显得生硬。使用 `useSpring` 进行插值计算，实现类弹簧行为，而不是立即更新。

```tsx
import { useSpring } from "framer-motion";

// 无弹簧效果：即时响应，感觉生硬
const rotation = mouseX * 0.1;

// 有弹簧效果：具有动量，感觉自然
const springRotation = useSpring(mouseX * 0.1, {
  stiffness: 100,
  damping: 10,
});
```

仅将此用于**装饰性**交互。如果是银行应用中的功能性图表，不使用动画反而更好。

## 使用瞬移而非弹簧

如果交互需要即时响应或精确计时，完全跳过弹簧动画。使用短时过渡或直接瞬移到最终状态。

```tsx
<motion.div
  animate={{ opacity: isOpen ? 1 : 0, x: isOpen ? 0 : -12 }}
  transition={
    shouldSnap
      ? { duration: 0.12, ease: "linear" }
      : { type: "spring", stiffness: 500, damping: 40 }
  }
/>
```
