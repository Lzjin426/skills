# 群聊身份规则

> 警告：群组操作中最常见的失败原因是选择了错误的身份。执行操作前请确认身份。

群聊操作同时支持 `--as user`（UAT 用户身份）和 `--as bot`（TAT 机器人身份）。选择正确的身份是成功的关键。

## 基本原则

- **如果用户明确指定了身份：** 严格使用用户请求的身份（`--as user` 或 `--as bot`），不要猜测。
- **如果用户未指定身份：** 根据上下文推断正确的身份，而不是依赖默认值。

## 按操作选择身份

| 操作 | 推荐身份 | 原因 |
|------|---------|-----------------------------------|
| 创建群组 (`+chat-create`) | 取决于场景 | 根据上下文推断 |
| 添加成员（成员管理流程） | `--as user` | 机器人可见性有限，当目标用户与机器人相互不可见时经常失败 (232024) |
| 更新群组 (`+chat-update`) | 群主身份 | 权限变更需要群主/管理员权限；群主转让需要群主身份 |

## 推断群主

当需要群主级别的操作且群主未知时，按以下顺序推断：

1. 机器人创建了群组且 **未** 指定 `--owner` -> 群主是机器人 (`--as bot`)
2. 机器人创建了群组且 **指定了** `--owner ou_xxx` -> 群主是该用户 (`--as user`)
3. 用户创建了群组且 **未** 指定 `--owner` -> 群主是当前用户 (`--as user`)
4. 仍不明确 -> 在进行群主级别更改前，请用户确认谁是群主

### 当群主既非当前用户也非机器人时

如果查询显示群主是第三方用户（`owner_id` 既不是当前授权用户也不是机器人），则当前身份没有群主权限。在这种情况下：

- **权限/设置更改：** 如果机器人是群组的管理员，`--as bot` 仍可以执行管理员级别的操作，例如重命名群组或更改权限。
- **仅群主可执行的操作（如群主转让）：** 需要实际群主通过 `lark-cli auth login` 完成 UAT 授权，然后以该群主身份执行操作。
- 向用户清楚地解释限制，而不是盲目重试。

## 常见陷阱

### 在群组创建时邀请成员

如果机器人创建群组且 `--users` 包含与机器人相互不可见的用户，整个请求将失败并返回 232043。请改用两步法：

1. 首先用机器人创建群组，排除不可见用户：`lark-cli im +chat-create --name "群组名称"`
2. 稍后使用用户身份的成员管理流程添加用户

### 权限不足

- **232016 / 232002 / 232017：** 当前身份不是群主或管理员 -> 切换到群主身份
- **232011：** 当前用户不在群组中 -> 使用群组成员身份，或先加入群组
- **232024：** 机器人与目标用户相互不可见 -> 切换到 `--as user`
