# Obsidian 的 JSON Canvas 规范

版本 1.0 — 2024-03-11

## 概述

JSON Canvas 是一种用于表示无限画布文档的格式。本规范定义了创建与 Obsidian 兼容的画布文件的结构。

## 顶层结构

根 JSON 对象包含两个可选数组：

```json
{
  "nodes": [...],
  "edges": [...]
}
```

- `nodes` (可选，数组): 所有画布对象（文本、文件、链接、组）
- `edges` (可选，数组): 节点之间的所有连接

## 节点类型

### 通用属性

所有节点共享以下必需属性：

- `id` (必需，字符串): 节点的唯一标识符
- `type` (必需，字符串): 节点类型 (`text`, `file`, `link`, `group`)
- `x` (必需，整数): X 坐标位置（像素）
- `y` (必需，整数): Y 坐标位置（像素）
- `width` (必需，整数): 宽度（像素）
- `height` (必需，整数): 高度（像素）
- `color` (可选，字符串/数字): 颜色（十六进制 `"#FF0000"` 或预设值 `"1"`）

### 文本节点

存储带有 Markdown 格式的纯文本。

**必需属性：**
- `text` (字符串): Markdown 语法内容

**示例：**
```json
{
  "id": "abc123",
  "type": "text",
  "x": 0,
  "y": 0,
  "width": 250,
  "height": 100,
  "text": "# 主要主题\n\n关键点在此",
  "color": "4"
}
```

### 文件节点

引用其他文件或附件（图像、PDF 等）。

**必需属性：**
- `file` (字符串): 库中文件的路径

**可选属性：**
- `subpath` (字符串): 指向特定标题/块的链接（以 `#` 开头）

**示例：**
```json
{
  "id": "def456",
  "type": "file",
  "x": 300,
  "y": 0,
  "width": 400,
  "height": 300,
  "file": "Images/diagram.png"
}
```

**带子路径：**
```json
{
  "id": "ghi789",
  "type": "file",
  "x": 0,
  "y": 200,
  "width": 250,
  "height": 100,
  "file": "Notes/Meeting Notes.md",
  "subpath": "#Action Items"
}
```

### 链接节点

引用外部 URL。

**必需属性：**
- `url` (字符串): 包含协议的完整 URL

**示例：**
```json
{
  "id": "jkl012",
  "type": "link",
  "x": 0,
  "y": -200,
  "width": 250,
  "height": 100,
  "url": "https://obsidian.md",
  "color": "5"
}
```

### 组节点

用于组织相关节点的可视化容器。

**可选属性：**
- `label` (字符串): 组的文本标签（推荐）
- `background` (字符串): 背景图像的路径
- `backgroundStyle` (字符串): 图像渲染样式
  - `cover`: 填充整个节点
  - `ratio`: 保持宽高比
  - `repeat`: 平铺为图案

**示例：**
```json
{
  "id": "group1",
  "type": "group",
  "x": -50,
  "y": -50,
  "width": 600,
  "height": 400,
  "label": "核心概念",
  "color": "4"
}
```

**带背景：**
```json
{
  "id": "group2",
  "type": "group",
  "x": 700,
  "y": 0,
  "width": 500,
  "height": 600,
  "label": "参考资料",
  "background": "Images/texture.png",
  "backgroundStyle": "repeat"
}
```

## Z 轴顺序与图层

节点按数组顺序显示：
- **第一个节点**: 底层（在其他节点下方渲染）
- **最后一个节点**: 顶层（在其他节点上方渲染）

**最佳实践顺序：**
1. 组节点（背景）
2. 子组
3. 常规节点（文本、文件、链接）

这确保了组出现在内容后面。

## 边（连接）

边用线条连接节点。

**必需属性：**
- `id` (必需，字符串): 唯一标识符
- `fromNode` (必需，字符串): 起始节点 ID
- `toNode` (必需，字符串): 结束节点 ID

**可选属性：**
- `fromSide` (字符串): 起始边侧
  - 值: `top`, `right`, `bottom`, `left`
- `fromEnd` (字符串): 起始端点形状
  - 值: `none` (默认), `arrow`
- `toSide` (字符串): 结束边侧
  - 值: `top`, `right`, `bottom`, `left`
- `toEnd` (字符串): 结束端点形状
  - 值: `arrow` (默认), `none`
- `color` (字符串/数字): 边颜色
- `label` (字符串): 边上的文本标签

**示例 - 简单连接：**
```json
{
  "id": "edge1",
  "fromNode": "abc123",
  "toNode": "def456"
}
```

**示例 - 完整指定：**
```json
{
  "id": "edge2",
  "fromNode": "def456",
  "fromSide": "bottom",
  "fromEnd": "none",
  "toNode": "ghi789",
  "toSide": "top",
  "toEnd": "arrow",
  "color": "3",
  "label": "导向"
}
```

## 颜色系统

### 预设颜色

使用字符串数字 `"1"` 到 `"6"`：

- `"1"` - 红色
- `"2"` - 橙色
- `"3"` - 黄色
- `"4"` - 绿色
- `"5"` - 青色
- `"6"` - 紫色

**注意：** 确切颜色会适配 Obsidian 的主题。这些预设颜色在亮/暗模式下提供语义含义。

### 自定义十六进制颜色

使用十六进制格式：`"#RRGGBB"`

**示例：**
- `"#4A90E2"` (蓝色)
- `"#50E3C2"` (青色)
- `"#F5A623"` (橙色)

**最佳实践：** 在同一个画布内使用一致的格式（全部十六进制或全部预设值）。

## 验证要求

创建画布文件时，请确保：

1.  **唯一 ID**：所有 `id` 值在节点和边中必须是唯一的
2.  **有效引用**：所有边的 `fromNode` 和 `toNode` 必须引用已存在的节点 ID
3.  **必需字段**：每种类型的所有必需属性都存在
4.  **有效坐标**：所有位置/尺寸值都是整数
5.  **颜色格式**：颜色使用十六进制 (`"#RRGGBB"`) 或预设字符串 (`"1"` 到 `"6"`)
6.  **引号转义**：JSON 字符串中的特殊字符已正确转义

## 常见问题与解决方案

### 问题：画布无法在 Obsidian 中打开
**解决方案：**
- 验证 JSON 语法（使用 JSON 验证器）
- 检查所有 ID 是否唯一
- 验证所有边引用是否存在
- 确保必需字段存在

### 问题：节点显示重叠
**解决方案：**
- 增加坐标之间的间距
- 在定位时考虑节点尺寸
- 使用最小间距：水平 320px，垂直 200px

### 问题：组未正确显示
**解决方案：**
- 确保组在数组中出现在内容节点之前
- 为所有组添加明确的 `label`
- 检查组尺寸是否包含子节点

### 问题：颜色不符合预期
**解决方案：**
- 使用一致的颜色格式（全部十六进制或全部预设值）
- 记住预设值会适配主题
- 如果使用自定义颜色，请在亮暗模式下都进行测试

### 问题：文本显示被截断
**解决方案：**
- 增加节点尺寸
- 将长文本拆分为多个节点
- 对于冗长内容，使用文件节点

## 中文内容的字符编码

当画布包含中文文本时，应用以下转换：

- 中文双引号 `"` → `『』`
- 中文单引号 `'` → `「」`
- 英文双引号必须转义：`\"`

**示例：**
```json
{
  "text": "『核心概念』包含:「子概念A」和「子概念B」"
}
```

这可以防止混合语言内容导致的 JSON 解析错误。

## 性能考虑

- **大型画布**：保持节点数量合理（<500 以获得流畅性能）
- **图像文件**：对背景使用压缩图像
- **文本长度**：保持节点文本简洁；对于长内容使用文件节点
- **边复杂度**：最小化交叉边以提高清晰度
