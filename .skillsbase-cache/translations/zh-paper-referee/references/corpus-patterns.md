# 样本文风与论证共性

本文件基于五篇中文工科论文样本提炼，不复制原文，而是抽取其稳定写法。适用于机械、制造、检测、控制、算法建模等“问题-方法-实验”型论文。

## 1. 摘要的稳定结构

高频结构是四句链条：

1. 交代工程背景或直接指出当前痛点
2. 给出“为此/针对上述问题，提出……”的核心方法
3. 概括方法由哪些关键模块组成
4. 用定量结果收束，说明效果与应用价值

摘要应当回答四个问题：
- 为什么要做
- 做了什么
- 怎么做的
- 结果如何

常见失误：
- 背景过长，方法只有一句
- 只有“提出方法”，没有关键机制
- 只有“效果良好”，没有定量指标
- 把意义写得比结果更大

## 2. 前言的推进方式

样本中最稳定的前言不是平铺直叙，而是递进式收缩：

1. 应用场景与工程重要性
2. 现有方法分类型回顾
3. 每类方法的具体不足
4. 当前研究缺口
5. 本文对准该缺口的解决方案

有效前言必须把“已有研究很多”推导成“本文仍有必要”。如果读完前言仍然不能回答“为什么必须是这篇文章”，说明缺口没有推出。

前言末段常见合格写法：
- 概述本文提出的方法
- 概述方法的关键组成
- 概述全文结构或主要贡献

## 3. 方法章节的组织方式

方法章节通常遵循“总述先行，模块拆解随后”的结构：

1. 总体框架/系统结构/问题建模
2. 子模块或子机制逐项展开
3. 关键公式、变量定义、约束条件
4. 模块之间如何衔接
5. 复杂度、控制策略或实现细节

样本的共同点不是公式多，而是每个公式都在解释链条里有位置：
- 公式前说明为什么要引入
- 公式后解释变量和物理/算法含义
- 后文试验回到这些设计是否有效

## 4. 实验章节的有效写法

高质量实验段落一般同时回答：
- 在什么环境下验证
- 与谁比较
- 用什么指标判断
- 结果具体是多少
- 为什么会得到该结果

常见合格证据包括：
- 与现有方法的客观指标对比
- 消融试验或组件贡献分析
- 鲁棒性/泛化性/多场景验证
- 仿真与实测的相互支撑

常见硬伤：
- 只给结果图，不解释机制
- 只比最弱基线，不比强基线
- 只选对自己有利的指标
- 把“一个场景可行”写成“普遍适用”

## 5. 结论的回收方式

样本结论基本都先总述，再分点回收贡献，最后做克制展望。

建议结论顺序：

1. 回答本文解决了什么问题
2. 分点对应主要贡献
3. 用关键结果支撑贡献
4. 给出边界清楚的应用意义或后续方向

避免：
- 在结论引入正文没有证明的新性能
- 把工程前景写成既成事实
- 重复摘要原句，不做贡献回收

## 6. 中文工科表达偏好

推荐：
- “针对……问题，提出……方法”
- “构建……模型/设计……模块/建立……策略”
- “试验结果表明……”
- “与……相比，……提升/降低……”
- “验证了所提方法的有效性/准确性/可行性”

慎用：
- “创新性极强”
- “显著领先所有方法”
- “完全解决了”
- “具有颠覆性意义”

## 7. 本 skill 默认模仿的不是词汇，而是约束

真正要学的是以下约束，而不是若干套话：
- 每个贡献都要有正文位置和证据来源
- 每个结果都要有对照对象或评价基准
- 每个章节都要服务总问题，不做无关铺陈
- 每句强结论都要能回指方法或数据
