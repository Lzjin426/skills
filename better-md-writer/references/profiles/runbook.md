# Profile: Runbook

## When to use

Use this profile for operational procedures, deployment steps, incident response, configuration actions, and other execution-critical workflows.

## Reader expectation

The reader expects exact steps, branching conditions, and a reliable way to verify completion.

## Default structure

- purpose and scope
- prerequisites
- ordered steps
- branch or failure handling
- verification and rollback when needed

## Default carrier

List-led, supported by short paragraphs or code blocks.

## Default passes

- research: `off`
- cognitive-path: `off`
- illustration: `optional`
- humanization: `off`

## Style defaults

- optimize for execution accuracy over narrative flow
- keep steps explicit and ordered
- do not tutorialize or generalize unless it prevents a concrete mistake
