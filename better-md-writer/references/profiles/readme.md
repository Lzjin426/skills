# Profile: README

## When to use

Use this profile for repository, project, tool, or package orientation documents.

## Reader expectation

The reader wants to understand what this is, why it matters, and how to get started fast.

## Default structure

- what this project is
- quick start or installation
- basic usage
- important constraints, pointers, or next links

## Default carrier

Mixed, usually paragraph-led with short code blocks and short lists.

## Default passes

- research: `off`
- cognitive-path: `off`
- illustration: `optional`
- humanization: `optional`

## Style defaults

- lead with orientation and getting-started value
- follow existing repo style when one already exists
- do not force numbered headings or top-level separators unless the project already uses them
