# Better Md Writer Core: Reader Check

Run this check before final delivery.

## Five required questions

1. Can the reader quickly tell why this document exists?
2. Does the structure match the document's job?
3. Is anything important assumed but never stated?
4. Is any section filler that adds shape without information?
5. Will the reader know what to do, decide, or understand next?

If any answer is weak, revise the draft before finishing.

## Optional pass checks

Only run the checks that match enabled passes:

- **Research enabled**
  - Were facts integrated into the document's own voice instead of pasted raw?
  - Did the research change or strengthen the draft in a meaningful way?

- **Cognitive path enabled**
  - Is the mental model explicit and useful?
  - Does the section order actually help understanding build over time?

- **Illustration enabled**
  - Does the visual remove real explanation cost?
  - Is the document still understandable without over-explaining the same thing twice?
  - Was there a separate illustration review, or were visuals decided only ad hoc during drafting?
  - Does each visual have a clear job, rather than existing because "a picture felt nice to have"?
  - If the document met mandatory illustration-review triggers, is there an explicit image plan?

- **KB article**
  - Was the article subtype clear before drafting: concept explanation, comparison, result interpretation, or method/process?
  - Does the article answer one explicit judgment question instead of only collecting facts?
  - Is there at least one real or planned visual anchor?
  - Can the reader picture what the thing looks like, not just repeat the definition?
  - Are technical terms explained in plain language before symbols, variable names, or implementation details take over?
  - Would merging thin sections make the article easier to follow?
  - Does the article stay professional and neutral, without slipping into spoken teaching cues or slogan-like headings?

- **Humanization enabled**
  - Did the cleanup reduce mechanical phrasing without weakening precision, scope, or traceability?

## Final rule

If a pass did not improve the document, remove its effects instead of defending the process.
