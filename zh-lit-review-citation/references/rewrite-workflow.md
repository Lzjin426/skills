# 改写工作流

## Step 1: 识别输入约束

- 提取待改写段落。
- 提取现有参考文献编号与题名。
- 判断用户要求是：
  - 重写综述
  - 只补引用
  - 调整引用位置
  - 压缩或扩写综述

## Step 2: 先定段落角色

只选一个主角色：

- 背景段
- 分类段
- 代表工作段
- 不足段
- 承接本文段
- 方法补引段
- 实验补引段

一段只承担一个主角色，最多兼带一个次角色。

## Step 3: 建立句子级证据链

对每一句做最小化标注：

| 句子功能 | 是否需要引文 | 对应参考文献 |
| --- | --- | --- |
| 背景事实 | 是 | 1-2 篇 |
| 分类总述 | 是 | 2-4 篇 |
| 代表工作 | 是 | 1 篇或少量 |
| 不足判断 | 通常是 | 1-3 篇 |
| 本文贡献 | 否 | 无 |

如果某句找不到可靠文献：

- 改成弱判断。
- 或删除。
- 或拆到前文已有证据的句子之后。

## Step 4: 按模板重写

### 模板 A：方法型引言综述段

1. 任务/场景价值。
2. 现有方法主要路线。
3. 一到两类代表工作。
4. 面向当前场景的具体不足。
5. 因此本文……

### 模板 B：分类综述段

1. 现有研究主要分为若干类。
2. 第一类的核心机制与代表工作。
3. 第二类的核心机制与代表工作。
4. 对比这些路线的差异与瓶颈。

### 模板 C：方法章节局部补引段

1. 本段要解决什么问题。
2. 借用或对照了什么经典理论/方法。
3. 本文如何改造或组合。

### 模板 D：实验章节补引段

1. 对比对象或数据集来源。
2. 指标定义或使用理由。
3. 本文结果解读。

第三句通常不引。

## Step 5: 出口检查

- 有没有虚构来源？
- 有没有“一个引用覆盖整段”？
- 有没有把“本文贡献句”也加上引文？
- 有没有用过强的结论词但没有证据？
- 有没有把多条技术路线混写在一段里？

## 用脚本做预筛查

在人工改写前，可先运行：

```bash
python scripts/review_citation_audit.py draft.md
```

它适合先找可疑位置，不适合替代人工学术判断。
