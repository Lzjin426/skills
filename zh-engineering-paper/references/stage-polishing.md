# 阶段 C：润色阶段 SOP

用户已有完整初稿，现在要润色到"投稿质量"。本 SOP 给出**三遍润色法**：每一遍只关注一类问题，不要试图一次改完所有问题（必然顾此失彼）。

## 润色总流程

```
第一遍：语言润色（改词、改句式）
   ↓
第二遍：逻辑润色（改段落、改论证）
   ↓
第三遍：形式润色（图表、公式、引用）
   ↓
终审：通读，反 AI 味检查
```

每一遍都用对应的清单。

---

## 第一遍：语言润色

目标：让每一句话像论文，而不是像综述报告 / 网文 / AI 生成。

### 1.1 替换 AI 高频词

中文 AI 生成内容的"指纹词汇"，看到先警惕：

| AI 高频 | 改为 |
|---|---|
| 旨在 | 删除（直接陈述目的） |
| 致力于 | 删除或改"本文聚焦于" |
| 探索性地 | 删除 |
| 一定程度上 | 删除或改具体程度 |
| 有效地 | 通常可删 |
| 显著地 | 用具体数字替换 |
| 进一步 | 视情况删除 |
| 综上所述 | 删除（除非真有总结性段） |
| 总而言之 | 删除 |
| 值得注意的是 | 删除 |
| 众所周知 | 删除（通常这后面跟的是不那么"众所周知"的内容） |
| 在当今社会 | 删除 |
| 随着 ... 的不断发展 | 改具体描述 |
| 全面提升 | 给出具体提升的指标 |
| 深度赋能 | 删除（这是 PPT 词，不是论文词） |
| 智能化（作为修饰词） | 通常可删，留具体功能 |
| 数字化（作为修饰词） | 同上 |

**判断方法**：删掉这个词，句子意思是否完整？是 → 删。

### 1.2 改被动堆叠

中文工科论文用主动 > 被动。"被发现""可以被""能够被" 连续出现 ≥ 2 次 → 改主动。

❌ "通过实验可以被验证，本方法的有效性能够被证明。"
✅ "实验验证了本方法的有效性。"

### 1.3 替换大词为具体词

| 大词 | 视情况换为 |
|---|---|
| 大量的 | "数千个"、"超过 N 组" |
| 许多 | 给具体数 |
| 显著的 | "提升 X%" |
| 明显的 | 同上 |
| 良好的 | 给具体性能值 |
| 较高的 | 数值或排名 |
| 较好的 | 同上 |
| 较少 | 数值 |

### 1.4 改长句

判断指标：
- 单句 > 80 字 → 警惕
- 单句 > 100 字 → 必须拆
- 定语 > 3 层 → 拆

拆句方法：
1. 找句中的分号/逗号断点
2. 把断点改为句号或者"其中"开头的新句

❌ "本文针对动态载荷下的双模量材料结构在传统位移单元易产生剪切锁定问题且无法准确描述复杂应力状态导致优化结果不可靠的难题，提出了一种基于混合应力单元和正交异性插值模型的拓扑优化方法。"

✅ "传统位移单元在双模量材料中易产生剪切锁定，且无法准确描述复杂应力状态，导致优化结果不可靠。针对这一问题，本文提出基于混合应力单元和正交异性插值模型的拓扑优化方法。"

### 1.5 替换口语词

| 口语 | 书面 |
|---|---|
| 这样 / 那样 | 视情况删或具体描述 |
| 的话 | 删除 |
| 就是 | 删除 |
| 其实 | 删除 |
| 比较好 | 给指标 |
| 不错 | 同上 |
| 一些 | 给数 |
| 很多 | 给数 |
| 一下 | 删除 |
| 然后 | "随后"、"接着"、"进而" |
| 但是 | "然而"、"但" |

### 1.6 标点规范

- 中文用全角标点：，。：；！？""''
- 英文/公式用半角标点
- 中英文之间的连接：根据期刊指南，多数期刊**不加空格**
- 数字与单位之间一般空格："5 mm" 而非 "5mm"
- 顿号"、"用于并列名词；不要用"，"代替顿号

### 1.7 术语统一

通读全文，建一份术语表。**一个概念在全文只用一种表达**。

常见混用：
- "剩余使用寿命 / 剩余寿命 / RUL" → 选 1
- "深度神经网络 / 深度网络 / DNN" → 选 1
- "卷积神经网络 / 卷积网络 / CNN" → 选 1（首次定义后用缩写）

英文缩写：第一次出现给中文 + (英文全称, 缩写)，之后用缩写。

### 1.8 第一遍清单

- [ ] AI 高频词全部检查替换
- [ ] 没有连续 ≥ 3 个被动句
- [ ] 大词都给了具体数字 / 具体描述
- [ ] 长句拆到 ≤ 80 字
- [ ] 口语词全清掉
- [ ] 标点统一全角
- [ ] 术语全文统一

## 第二遍：逻辑润色

目标：让段落之间、论证之间的关系清晰。

### 2.1 段落功能审查

每一段问自己：**这段在论证什么？**

如果答不上来 → 这段没有功能 → 删 / 改 / 合并。

每段功能可能是：
- 提出问题
- 介绍现有工作
- 指出现有工作的不足
- 提出本文方法
- 描述方法细节
- 给出实验结果
- 解读实验结果
- 总结本节工作

一段不要承担多个功能（除非是过渡段）。

### 2.2 段落首句

每段第一句应该**总领段意**（topic sentence）。读者只读首句也能跟上论证主线。

❌ 段首："Y 数据集来自公开仓库 [N]。"
   段中："本文使用 X、Y、Z 三个数据集。"

✅ 段首："本文使用 X、Y、Z 三个数据集。"
   段中："Y 数据集来自公开仓库 [N]。"

### 2.3 段落之间的过渡

段落之间应该有**显式的过渡词或过渡句**：

| 关系 | 过渡词 |
|---|---|
| 因果 | 因此、于是、由此 |
| 转折 | 然而、但、不过 |
| 递进 | 进一步、更重要的是、在此基础上 |
| 并列 | 此外、另外、与此同时、相应地 |
| 对比 | 相较之下、与之相反、不同于 |
| 总结 | 综上、总之（仅在确实总结的位置用） |

注意"综上所述"、"总而言之"在中文工科论文中用得不多，**滥用是 AI 味的标志**。

### 2.4 论证链审查

引言、方法的"动机段"、结论这三处，是论证最密集的地方。每处审查：

```
要论证的结论：________________
论据 1：______________ → 是否充分？
论据 2：______________ → 是否充分？
论据 3：______________ → 是否充分？
推论： 上述论据 → 结论 是否成立？
```

常见问题：
- 论据是别人的工作，但用别人的工作论证自己的方法 → 逻辑跳跃
- 论据是定性的，但结论是定量的 → 论据不足
- 论据不是因果关系，是相关关系 → 表述要谨慎

### 2.5 引言到方法到实验的"穿线"

三处必须互相呼应：

| 引言末段 | 方法章节 | 实验章节 |
|---|---|---|
| 创新点 (1) | 第 2 章 | 主实验 + 消融第 1 项 |
| 创新点 (2) | 第 3 章 | 主实验 + 消融第 2 项 |
| 创新点 (3) | 第 4 章 | 主实验 + 消融第 3 项 |

**对照检查**：每个创新点都能找到对应的方法章节和实验验证？找不到 → 创新点没落实 → 改。

### 2.6 第二遍清单

- [ ] 每段都能说清"在论证什么"
- [ ] 每段首句都是 topic sentence
- [ ] 段落之间有过渡（避免"综上所述"滥用）
- [ ] 引言/方法/结论的论证链清晰，无跳跃
- [ ] 引言创新点 ↔ 方法章节 ↔ 实验验证三处呼应

## 第三遍：形式润色

目标：让图表、公式、引用、格式整齐。

### 3.1 图表检查

→ 详见 `references/figures.md` 自查清单

每张图：
- [ ] 中文宋体，英文 Times New Roman
- [ ] 字号符合层级（标题 ≥ 标签 ≥ 刻度）
- [ ] 配色 ≤ 5 种主色，色弱友好
- [ ] 坐标轴单位齐全（"量名 / 单位"）
- [ ] 图例位置合理
- [ ] 图标题在图下方
- [ ] 正文有引用并解读

每张表：
- [ ] 三线表（无竖线）
- [ ] 表头加粗
- [ ] 数值列右对齐，小数位统一
- [ ] 单位放在表头
- [ ] 最优值加粗、次优加粗（不下划线就用一种区分）
- [ ] 表标题在表上方

### 3.2 公式检查

- [ ] 重要公式编号；同章公式连续编号
- [ ] 矩阵/向量加粗
- [ ] 标量斜体；常量、单位正体
- [ ] 同一物理量全文同一符号
- [ ] 首次出现的公式有"其中"开头的符号说明
- [ ] 公式前后有文字过渡（不孤立悬空）

### 3.3 引用检查

- [ ] 每个 [N] 在参考文献列表都能对应
- [ ] 参考文献列表无重复
- [ ] 引用顺序与正文出现顺序一致（按"按出现顺序"格式期刊）
- [ ] 引用格式符合目标期刊指南
- [ ] （参考文献的细节由用户的另一个 skill 处理）

### 3.4 排版细节

- [ ] 章节标题层级一致（1 / 1.1 / 1.1.1）
- [ ] 章节编号连续无跳号
- [ ] 没有"orphan line"（一段最后只有一个字单独成行）
- [ ] 段落首行缩进 2 字符（中文规范）
- [ ] 没有半角逗号、句号在中文段落里
- [ ] 数字和单位之间一致（要么都空格、要么都不空格）

### 3.5 第三遍清单

- [ ] 所有图表通过 figures.md 自查
- [ ] 公式格式规范
- [ ] 引用编号正确连续
- [ ] 排版无明显瑕疵

## 终审：反 AI 味通读

最后一步：**用人的语感**通读全文（或一节一节读），找出"看上去像 AI 写的"段落。

### AI 味的常见特征

#### 特征 1：结构癖

任何东西都"首先...其次...最后..."。变量稍多就"在 X 方面、在 Y 方面、在 Z 方面"。

修正：变换连接方式。用"一是...二是...三是..."、"主要包括..."、"分别从 X、Y、Z 三个方面..."、或者干脆不分点直接铺陈。

#### 特征 2：形容词堆砌

"广泛的""深入的""系统的""全面的"作为前置形容词。一句话里出现 3 个以上 → 删。

#### 特征 3：句长一致

每句话长度都差不多（25–35 字）。AI 喜欢中等长度。**人的写作长短交错**——短句点睛，长句论证。

修正：每段刻意加 1 个 < 15 字的短句作为节奏调节。例：

> "本文方法在 NASA 数据集上达到 12.7 RMSE，相比 Transformer 基线降低 21.6%。性能提升显著。这一改进主要来自……"
（"性能提升显著"是短句节奏调节）

#### 特征 4：信息密度均匀

AI 写的段落信息密度均匀（每句话承载差不多的信息量）。人写的段落有详有略——主论点详写，背景一句带过。

#### 特征 5：完美无个性

人写的论文偶尔有"小毛病"——某个例子不那么贴切、某个连接稍硬、某个转折稍突兀。AI 写的过分流畅。

修正：不必刻意制造毛病，但也不必把"看上去稍硬但意思清晰"的句子改成"流畅但空洞"的版本。

#### 特征 6：模板化的开头

"在当今社会..."、"随着...的不断发展..."、"近年来..."——这是中学议论文/AI 的开头模板。中文工科论文开头应该**直接定义领域**或**直接讲应用场景**。

#### 特征 7：避免争议的中庸

AI 不愿下判断。"在某种程度上""有一定的""可能存在"——读起来什么都说了又什么都没说。

修正：要么给具体证据下判断，要么干脆不写。

### 终审清单

- [ ] 没有结构癖（不是所有段都"首先...其次..."）
- [ ] 没有形容词堆砌
- [ ] 句长有节奏（短中长交错）
- [ ] 信息密度有详有略
- [ ] 开头不是"在当今社会..."这种模板
- [ ] 不用"在某种程度上"这种避免下判断的虚词

## 一些"立竿见影"的快速润色技巧

如果用户时间紧，没法做完三遍，按优先级排：

**P0（必做）**：
1. 删除 AI 高频词（1.1）
2. 替换大词为具体数（1.3）
3. 检查引言/方法/实验的创新点呼应（2.5）

**P1（强烈推荐）**：
4. 拆长句（1.4）
5. 段落首句（2.2）
6. 图表自查（3.1）

**P2（有时间再做）**：
7. 标点和术语统一（1.6, 1.7）
8. 排版细节（3.4）
