# 阶段 B：写作阶段 SOP

用户已有大纲、创新点、技术路线，现在要落笔写每一章。本 SOP 按章节给出**逐节模板**和**必填字段清单**。

**核心理念**：先把"信息骨架"搭出来，再填充语言。不要一开始就追求文采——文采在润色阶段处理。

## 总则：写作的工作模式

### 推荐写作顺序

不是按章节顺序写。推荐的顺序：

```
1. 方法章（你最熟悉，先写）
2. 实验章（依赖方法章已定型）
3. 引言（依赖创新点的最终表达）
4. 结论（依赖实验数据）
5. 摘要（最后写）
6. 标题（迭代调整）
```

理由：方法/实验是论文的"骨头"。骨头确定，皮毛（引言、摘要、标题）才能精准写。

### Claude 协助的边界

skill 写论文的合适方式：
- ✅ **给骨架 / 模板** —— 用户填具体内容
- ✅ **改语言 / 调结构** —— 用户给草稿，skill 改
- ✅ **检查规范** —— 是否符合范式
- ❌ **凭空生成实验数据** —— 永远不要
- ❌ **替用户做研究判断** —— 该方法是否成立等
- ❌ **直接生成大段"看上去像论文"的文字** —— 容易 AI 味重

**给用户的话**：写作阶段最高效的协作模式是：用户提供研究内容，skill 提供结构和语言；不要让 skill"代写"。

## 写标题

→ 详见 `writing-patterns.md` 第 1 节"标题"

### 标题工作流

1. 用户提供研究内容描述（30–50 字）
2. 按三种结构（方法主导 / 工况主导 / 综述）各产出 2 个候选，共 6 个
3. 与用户讨论选定 1 个主选 + 1–2 个备选
4. 检查清单：
   - [ ] 16–28 字
   - [ ] 三层信息齐全（方法/特性 + 对象 + 场景）
   - [ ] 关键词在前
   - [ ] 不出现"研究"二字结尾（除非是综述）
   - [ ] 不出现"一种新颖的""一种有效的"等水词

## 写摘要

> 📖 **强烈建议先读** `references/exemplars.md` 的"A.2 摘要"和"B.2 摘要"两节，对照真实顶刊摘要的节奏。规则只是骨架，仿例是肉。

### 工作流

**先填表，再成文**。让用户填以下表格：

```
背景： __________________________ （1 句）
问题： __________________________ （1 句，必须有"然而""目前"等转折）
方法： 步骤 1：_____________________
       步骤 2：_____________________
       步骤 3：_____________________
结果： 验证手段：_____________________
       关键定量结果：________________ （必须有具体数字）
意义： __________________________ （"为...提供..."句式）
```

填完后串成 250–400 字的中文摘要。

### 摘要写法的 7 条规则

1. **第一句不出现"本文"**——用领域陈述开头，更专业
2. **问题段必须有转折词**——"然而"、"但"、"目前 ... 仍 ..."
3. **方法段用"首先...其次...接着...最后..."递进**——3–4 步最合适
4. **结果段必须有具体数字**——"提升 N%"、"降至 X"、"在 Y 数据集上..."
5. **意义段不空喊口号**——"为...提供..."的"..."要具体（理论支撑 / 设计依据 / 工程指导，三选一最合适）
6. **不出现引用编号** [1][2]（部分期刊禁止摘要引文）
7. **关键词 5–8 个**：3 个领域大词 + 3 个具体方法/对象词 + 1–2 个差异化词

### 摘要的"瘦身检查"

中文摘要常见的 3 类水分：

- **空洞修饰**："深入研究"、"系统分析"、"全面探索"——通常可删
- **重复信息**：摘要里说了"提出了 X 方法"，结果段又说"X 方法表明..."，第二个改成"实验/数值算例表明"
- **未验证陈述**：写"具有广阔应用前景"——除非论文里真有应用案例，否则删

## 写引言

> 📖 **强烈建议先读** `references/exemplars.md` 的"A.3 引言"四段（约 1100 字仿例 + 段落功能标注）。这是整个 skill 里最值得仿写的部分——4 段式骨架在仿例 A 中体现得最完整。

### 工作流

引言是最难写的部分。建议**分 4 次写**，每次只写一段：

**第 1 次：写第 1 段（领域定义 + 学术脉络）**

让用户提供：
- 领域名称（一个词，3–6 字）
- 1 位奠基人 + 1 个奠基方法
- 2–3 个子方向（每个一句话）

按模板（writing-patterns.md 中第 1 段模板）填空。

**第 2–N 次：写中段（现状梳理）**

按主流技术路线分段。每段：
- 选 2–4 篇代表作
- 用"XX 等 [N] 提出了 ... 通过 ... 实现了 ... 但 / 然而 ..."句式
- 每段最后一句必须**收紧到一个不足**

提醒用户：**每段只讲一个子话题**，不要把"方法 A 的工作"和"方法 B 的工作"堆在同一段。

**第 (N+1) 次：写末段（研究空白 + 本文工作）**

用 5 个研究空白句式（writing-patterns.md）选一个，然后用三段论：

```
然而，... 仍 ... / 鲜有 ... / 尚未 ...
为此，本文 ...
具体而言：
  (1) ...
  (2) ...
  (3) ...
本文结构如下 ...（可选）
```

### 引言常见错误

| 错误 | 修正 |
|---|---|
| 第 1 段堆砌应用例子 | 改为"领域定义 + 脉络"，应用例子放在意义段 |
| 现状梳理像参考文献流水账 | 按子话题组织，每段收紧到一个不足 |
| 中段对每篇文献评价过分细致 | 综述才需要细，研究论文每篇一两句话即可 |
| 末段创新点 (1)(2)(3) 之间无逻辑关系 | 创新点应递进或并列；标明它们如何对应方法章节 |
| "本文章""本文研究""本文工作"反复出现 | 改用"本文""本研究"中的一种，不混用 |
| 引言里出现实验数据 | 实验数据放实验章；引言只讲"做了什么"不讲"做得多好" |

### 引言字数控制

- 第 1 段：200–300 字
- 现状梳理：800–1500 字（按子话题数）
- 末段：300–500 字（创新点 3 条占大头）
- **总计 1500–2500 字**为佳；超过 3000 字考虑精简

## 写方法章

### 章节内部结构

每个方法章都按以下结构展开：

```
[X.0 引言段]      用 1–2 段说明本章解决什么问题、整体思路（80–150 字）
[X.1 问题形式化]  数学定义、变量、约束（如果整篇方法章只有一个问题，可放本章前）
[X.2 模块设计]    具体方法描述
   [X.2.1 动机]   为什么需要这个模块（50–100 字）
   [X.2.2 设计]   公式 + 文字解释（核心，500–1500 字）
   [X.2.3 性质]   复杂度 / 收敛性 / 边界条件（可选）
[X.3 模块设计 2]
[X.4 整体算法] 伪代码或流程图，展示模块如何串起来
```

### 方法章的"动机段"

每个核心创新模块前**必须**有 50–100 字的动机段。结构：

```
[现有问题]   传统方法在 ... 时存在 ... 问题。
[本文思路]   为此，本节提出 ...
[预期效果]   通过 ...，能够 ...
```

样本：
> 传统拓扑优化方法在双模量材料中因单元只能区分受拉或受压两种状态，无法描述复杂应力下的混合状态。为此，本节提出基于正交异性材料的四相插值模型。通过引入正交异性材料模拟单元同时受拉/压时的力学响应，能够更真实地刻画双模量材料在多工况载荷下的实际行为。

### 方法章的"图必备"

每个方法章应该至少有 1 张图，常见类型：
- **整体架构图**：放在本章首节
- **模块细节图**：每个核心模块一张
- **算法流程图**：放在末节

→ 详见 `references/figures.md`

### 公式书写规范

1. 重要公式编号 (X-Y)；中间过程不编号
2. 公式后必须有"其中"开头的符号说明（首次出现时）
3. 矩阵/向量加粗 **A**；标量不加粗
4. 标量、变量斜体；常量、单位、运算符正体
5. 同一物理量全文同一符号；中英变量不混用
6. 不要让公式独占一行而前后没有文字过渡

公式示意：

> 优化目标可以表示为
> 
> $$\min_{\boldsymbol{\rho}} f(\boldsymbol{\rho}) = \mathbf{u}^T \mathbf{K}(\boldsymbol{\rho}) \mathbf{u}  \quad (3\text{-}1)$$
> 
> 其中 $\boldsymbol{\rho}$ 为单元密度向量，$\mathbf{K}(\boldsymbol{\rho})$ 为整体刚度矩阵，$\mathbf{u}$ 为位移场。

## 写实验章

### 实验章结构

```
5 实验验证
   5.1 实验设置
       5.1.1 数据集
       5.1.2 评价指标
       5.1.3 对比方法
       5.1.4 实现细节
   5.2 主实验
   5.3 消融实验
   5.4 参数敏感性分析（可选）
   5.5 可视化分析
   5.6 计算复杂度分析（可选）
```

### 实验设置的"四件套"

#### 数据集
- 名称、来源（公开数据集要给引用）
- 规模（样本数、维度、采样率）
- 划分方式（训练/验证/测试比例；是否分层；是否考虑跨工况）
- 预处理方式（必须可复现）

#### 评价指标
- 每个指标给**定义公式**（首次出现）
- 单位
- 越大越好（↑）/ 越小越好（↓）
- 物理意义解释（不仅是数学定义）

#### 对比方法
- 选 4–7 个 baseline，分类列出：
  - 经典方法（领域内传统）
  - 近年代表性工作（顶会/顶刊）
  - 同期工作（必比）
- 每个 baseline 给一句话说明"它代表什么思路"
- 复现来源：作者代码 / 自己实现 / 文献报告

#### 实现细节
- 硬件：GPU 型号 / CPU / 内存
- 软件：框架版本（PyTorch X.Y / TensorFlow X.Y / MATLAB Rxxxx）
- 超参：学习率、batch size、迭代轮数、优化器
- 训练策略：早停、学习率调度、数据增强

**所有数字必须给具体值**。"我们设置了合适的学习率"——审稿人会要求重写。

### 主实验描述模板

```
[实验目的]   本节评估 ... 的性能。
[实验设置]   按 5.1 节设置进行 ... （此处不重复细节）
[结果展示]   表 X / 图 Y 给出了 ... 的对比结果。
[定量解读]   从表 X 可以看出，本文方法在 ... 指标上达到 ...，相比次优方法 ... 提升了 ...%。在 ... 指标上 ...
[机理解读]   这是由于本文方法 ...
[与现有最优方法对比]  相较于 ...
```

### 消融实验的写法

消融实验是审稿人最爱看的（证明你的每个模块都有用）：

```
[设置]   设计 N 个变体，逐项移除 / 替换核心模块。
[表格]   按变体 vs 模块 vs 性能格式
[逐项解读]
   - 去掉 A 后，性能下降 X%，说明 A 在 ... 起到关键作用
   - 去掉 B 后，... 
   - 完整模型 vs 任意单变体的差距均 > Y%，验证了模块组合的必要性
```

提醒用户：**消融的变体不要太多**（4–6 个）。变体 > 8 个会让审稿人困惑。

### 可视化分析

放 1–3 张能"看懂方法在做什么"的图：
- 注意力热力图
- 特征空间分布（t-SNE / UMAP）
- 中间层特征可视化
- 失败案例分析

→ 详见 `references/figures.md` 的图分类规则

## 写结论

> 📖 **强烈建议先读** `references/exemplars.md` 的"A.4 结论"和"B.5 结论"。仿例 A 用 ①②③ 圈号列具体观察是顶刊力学方向的典型写法；仿例 B 用 (1)(2) 写不足是 AI 方向的典型写法。

### 结构

```
[1]  本文针对 X 问题，提出了 ... 方法 / 框架。   （概述本文工作，1 句）
[2]  主要工作和成果可以总结为：
     (1) ... + 对应实验结果（量化）
     (2) ... + 对应实验结果
     (3) ... + 对应实验结果
[3]  本文的工作为 X 提供了 Y 方面的支撑 / 借鉴 / 思路。  （应用价值，1 句）
[4]  未来工作将关注 ...，以期 ...   （展望，可选，1–2 句）
```

### 与摘要、引言的对照检查

写完结论后，**回去对照**摘要和引言末段：

| 项 | 摘要 | 引言末段 | 结论 |
|---|---|---|---|
| 表述方式 | 简洁概括 | 明确分点 | 关联实验 |
| 数据 | 关键 1–2 个 | 不引数据 | 引关键数据 |
| 篇幅 | 50 字 / 创新点 | 80 字 / 创新点 | 60 字 / 创新点 |

三处的创新点表述应**一致但不重复**，从"提出 → 实现 → 验证"三个视角递进。

### 结论的常见错误

| 错误 | 修正 |
|---|---|
| 结论是摘要的复制版 | 加入实验定量结果，区别于摘要 |
| 罗列做过什么（"本文做了 ... 实验"） | 改为成果（"实验证明本文方法 ..."） |
| 写了"未来工作"但只是空话 | 要具体（"未来将拓展到 ... 工况"，而非"进一步研究") |
| 出现新概念 / 新方法 | 结论不应引入未在正文出现的内容 |
| 篇幅过长（> 800 字） | 砍到 300–500 字 |

## 写作阶段输出物

每章完成后应有：

- [ ] 完整章节文本（含公式、图表占位符）
- [ ] 章节内的图表清单（图 X-Y 标题，对应技术路线图哪一块）
- [ ] 章节摘要 1–2 句（用于结论回扣）

完成全文后：

- [ ] 标题 / 摘要 / 关键词
- [ ] 引言（4 段式）
- [ ] 方法（按创新点分章）
- [ ] 实验（含主实验、消融、可视化）
- [ ] 结论
- [ ] 致谢（可选）
