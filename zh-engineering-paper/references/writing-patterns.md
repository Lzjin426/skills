## 1. 标题

### 三类常用结构

| 结构 | 模板 | 真实样本 |
|---|---|---|
| **方法主导型** | 基于 X 的 Y 方法 / 基于 X 的 Y 研究 | "基于跨域层次聚合的轻量级工件表面图像超分辨率方法" |
| **工况主导型** | X 作用下的 Y 优化 / X 阶段 Y 的 Z 分析 | "动态载荷作用下的双模量材料结构拓扑优化"、"启动阶段风电齿轮箱行星轮径向-双向推力耦合滑动轴承瞬态润滑性能分析" |
| **综述型** | X 的 Y 机理与 Z 技术研究进展及挑战 | "燃油齿轮泵振动的动力学机理与减振技术研究进展及挑战" |

### 标题构造原则

1. **三层信息缺一不可**：核心方法/特性 + 作用对象 + 学科场景。例如"基于动力学特性优化（方法）的海上风电齿轮箱（对象+场景）配齿研究（任务）"。
2. **字数控制在 16–28 字**。少于 16 字常常信息量不足；超过 28 字会被审稿人/编辑要求精简。
3. **可量化就量化**："轻量级"、"高承载"、"高端"优于"小巧的"、"承载好的"、"先进的"。
4. **避免"研究"结尾的标题除非是综述或纯理论文章**。优先以"方法/分析/优化/识别/预测"等任务名结尾。
5. **关键词在标题前部**。审稿系统/读者扫读，前 6 字最重要。

### 反例

- ❌ "一种新颖的基于深度学习的故障诊断方法的研究"（"一种新颖的"、"...的研究"都是水词）
- ✅ "基于多尺度残差网络的旋转机械故障智能诊断"

## 2. 摘要

### 五段式骨架

中文工科顶刊摘要严格遵循以下结构。每段不必单独成行，但顺序不能乱：

```
[背景一句]      → 这个方向是干什么的、为什么重要
[问题一句]      → 现有方法不能解决什么 / 存在什么不足
[方法 2–4 句]   → 本文提出了什么，分几步实现
[结果 1–2 句]   → 验证手段 + 关键定量结果
[意义一句]      → 为...提供了...支撑/方法
```

总字数 **250–400 字**（中文核心常见区间）。

### 真实样本结构标注

**蔡金虎《动态载荷作用下的双模量材料结构拓扑优化》**：

> [背景] 实际工程中的很多材料在拉伸和压缩状态下具有不同的力学性能（即双模量特性）。[问题] 然而，针对这类材料结构的拓扑优化方法研究很少，尤其是极少考虑动态载荷下的双模量材料结构设计。[方法] 首先，采用混合应力单元离散设计域，以获得更准确的应力计算结果；其次，建立基于单元主应力符号和拉/压容限的单元拉/压状态判断准则，构建了四相材料插值模型实现双模量材料结构优化设计；接着，发展多工况载荷下基于单元拉/压应变能的单元拉/压状态判断准则，实现了多工况载荷下的材料分布优化。[结果] 通过典型数值算例验证了方法的有效性，并分析了拉/压弹性模量比值、激励频率等参数对优化构型的影响规律。[意义] 为双模量材料结构在动态载荷下的轻量化设计提供理论支撑。

### 摘要句式模板

| 段位 | 句式 |
|---|---|
| 背景 | "X 是 Y 的关键 / 核心内容。"、"X 在 Y 领域具有重要应用价值。"、"X 作为 Y 的关键部件/技术，..." |
| 问题 | "然而，..."、"目前，..."、"现有方法在...方面仍存在...不足。" |
| 方法引入 | "为此/因此，本文提出了...方法。"、"针对上述问题，本文..."、"从 X 的视角出发，..." |
| 方法分步 | "首先...；其次...；接着...；最后..." 或 "(1)...（2)...(3)..." |
| 结果 | "通过 X 验证了..."、"实验结果表明..."、"在 X 数据集上，所提方法将 Y 由 A 降至 B（降幅 N%）" |
| 意义 | "为...提供了..."、"为...奠定基础"、"对...具有指导意义" |

### 摘要的"三不写"

- **不写文献综述**。摘要不引文献、不评价他人。
- **不写未来工作**。摘要只讲本文做了什么。
- **不写技术细节的公式**。摘要中尽量不出现变量符号；要写也只用文字描述。

## 3. 引言

### 四段式骨架

```
[第 1 段] 定义本领域 + 学术脉络（200–300 字）
[第 2 段] 现状梳理：方向 1 的工作（按子话题切分，每个子话题一段）
[第 3 段] 现状梳理：方向 2、方向 3 ...
[第 N 段] 研究空白 + 本文工作 + 创新点（最重要的一段）
```

字数：1500–3000 字（按论文类型）。引言占全文比例约 10–15%。

### 第 1 段：定义领域与脉络

固定结构：**领域定义 → 奠基性工作 → 子方向列举 → 工程意义**。

样本（蔡金虎）：
> "拓扑优化作为一种结构轻量化设计方法，旨在通过优化材料分布，获得给定约束条件下的最优拓扑构型 [1]。自 BENDSØE 和 KIKUCHI 提出均匀化方法 [2] 以来，该领域研究范畴已从传统的柔度优化拓展至多物理场耦合问题。学者们在热力耦合拓扑优化 [3]、装配结构系统优化 [4] 等方向取得显著进展；同时，面向工程应用的大规模并行计算框架 [5] 和考虑增材制造工艺的多尺度结构设计 [6] 的发展，进一步推动了该领域的工程实用化进程。"

模板：
```
{领域名词} 作为 {上位概念}，{核心目的}[1]。
自 {奠基者} 提出 {经典方法}[2] 以来，
该领域研究已从 {经典任务} 拓展至 {前沿任务}。
学者们在 {子方向 A}[3]、{子方向 B}[4] 等方向取得显著进展；
同时，{工程方向}[5] 和 {新兴方向}[6] 的发展，
进一步推动了该领域的 {价值导向}。
```

### 中段：现状梳理（最难写）

**坑点**：很多人把现状梳理写成"流水账参考文献"——A 做了 a，B 做了 b，C 做了 c。审稿人不想看流水账，他想看**对比分析**。

正确做法：按**子话题/技术路线**组织，不是按时间顺序。每段只讲一个子话题。每个子话题内部用以下三种句式之一：

**句式 1：单项工作介绍**
> "XX 等 [N] 提出了 / 建立了 / 基于 X 方法，通过 Y 实现了 Z，但 / 然而 ..."

样本（张颖）：
> "JANG 等 [15] 基于等效静态载荷方法构建了时域动态响应拓扑优化框架；JIANG 等 [16] 面向多晶胞双尺度结构的时域动刚度最优设计问题，考虑不同晶胞间的可连接性，并行设计了微结构的构型及其宏观布局。"

**句式 2：对比式梳理**
> "相较于 X，Y 展现出 Z 优势，但仍面临 W 的局限。"

**句式 3：方向总结式**
> "在 X 领域，学者们在 Y 方向取得显著进展；然而，针对 Z 仍鲜有涉及。"

**关键技巧**：每个子话题的最后一句必须**收紧到一个不足**，为下一段或末段的"研究空白"做铺垫。不要光列别人做了什么——要指出别人**没做什么/做得不够好**。

### 末段：研究空白 + 本文工作

**这是引言最重要的一段**。结构：

```
[研究空白]   "然而，... 仍 ... / 鲜有 ... / 尚未充分考虑 ..."
[过渡]       "为此 / 针对上述问题，本文 ..."
[本文工作总述] "本文提出了一种 ..."
[创新点列举] "具体而言：(1) ...; (2) ...; (3) ..."
[文章结构]   "本文结构如下：第 1 节 ...；第 2 节 ..."（可选）
```

### 五个表达"研究空白"的固定句式

1. **"然而，... 仍面临 ... 的挑战"**
   > 然而，针对双模量材料结构在动态载荷下的拓扑优化设计仍面临诸多挑战。
2. **"... 鲜有涉及 ..."**
   > 上述研究只考虑了静态载荷条件下的结构响应问题，鲜有涉及动态载荷作用下的优化设计。
3. **"... 存在 ... 的局限 / 不足"**
   > 由于深度神经网络本质上具有黑箱特性，制约了模型决策的可解释性。
4. **"... 需要进一步研究"**
   > 如何聚焦输入时间窗口数据中的关键部位，并充分利用时序数据中隐含的信息需要进一步研究。
5. **"... 已成为该领域亟待解决的关键问题"**
   > 为此，深入探究 X 的振动产生机理与传递特性，进而有效抑制泵体振动已成为该领域亟待解决的关键问题。

## 4. 章节结构

### 方法/实验类论文

```
0. 引言
1. 问题描述（或：基础理论 / 预备知识）
2. 方法（核心创新点 1）
3. 方法（核心创新点 2）
4. 方法（核心创新点 3）—— 或合并为 2 章
5. 实验验证 / 数值算例
6. 结论
```

**特点**：方法分章节，每章对应一个创新点；实验单独成章。

### 综述类论文

```
0. 前言
1. {对象}的{机理/原理}建模         （讲清楚对象的物理/数学本质）
2. {子技术 1} 研究进展             （按技术路线梳理，每条路线一节）
3. {子技术 2} 研究进展
4. {子技术 N} 研究进展
5. 面临的挑战与机遇
6. 结论
```

**特点**：按"激励源 → 传递 → 控制"或"问题 → 方法 → 应用"的逻辑组织子节。"挑战与机遇"单独成节是顶刊综述的标志。

### 章节命名习惯

- 用名词短语，不用句子。✅ "方法验证"  ❌ "我们对方法进行了验证"
- 数字编号风格统一：要么"1.1.1 ..."，要么"1.1) ..."，全文一致
- 二级标题（X.X）字数 4–10 字最佳；三级标题（X.X.X）2–6 字
- 避免"概述""简介""分析"这种万能词单独做标题，要带上具体对象

## 5. 方法章节

### 写作骨架

```
[1] 问题形式化（数学描述）
    - 给定 X，目标是 Y
    - 列出符号、变量、约束
[2] 方法整体框架
    - 一张技术路线图
    - 一段话总览
[3] 模块 A 详细描述
    [3.1] 动机：为什么需要这个模块
    [3.2] 设计：具体怎么做（公式 + 文字）
    [3.3] 性质：复杂度 / 收敛性 / 边界条件
[4] 模块 B、C ...（同上）
[5] 整体算法流程
    - 伪代码或流程图
```

### 公式书写

- 重要公式编号；中间步骤可以不编号
- 公式后必须有"其中，X 表示 ..."的符号说明，第一次出现时给出
- 同一个量在全文用同一个符号；ASCII 字母（如 v）和希腊字母（如 ν）混用要小心
- 矩阵/向量加粗（**A**），标量不加粗
- 不出现没用过的符号

### 算法/方法名命名

- 论文中最好给方法起一个**3–5 字的中文简称**或**3–6 字的英文缩写**，方便后文反复引用
- 例：CDHA（Cross-Domain Hierarchical Aggregation，朱显丞）；DPGN-Bayes（张九思）
- 起名后第一次定义，全文统一使用

## 6. 实验章节

### 标准结构

```
1. 实验设置
   - 数据集：来源、规模、划分方式
   - 评价指标：每个指标的定义、单位、越大越好/越小越好
   - 对比方法：列出 baseline，说明每个的代表性
   - 实现细节：硬件、软件、超参（学习率、batch size、迭代轮数）
2. 主实验
   - 表格 + 关键定量结果
   - 文字解读："本文方法在 X 指标上达到 Y，相比次优方法 Z 提升了 N%"
3. 消融实验
   - 逐项去掉/替换某个模块，观察性能变化
   - 表格清晰对应每个变体
4. 参数敏感性分析（可选）
5. 可视化分析
   - 把模型/方法的"内部"展示出来（特征图、注意力、收敛曲线）
6. 与现有最优方法对比讨论
```

### 实验结果的描述句式

- ✅ "在 NASA 涡扇发动机数据集上，本文方法将 RMSE 由 16.2 降至 12.7（降幅 21.6%）"
- ✅ "相比 Transformer 基线，本文方法在参数量减少 38% 的条件下取得了相当的性能"
- ❌ "本文方法效果优秀"
- ❌ "实验结果令人满意"

**任何实验描述都必须有：指标值 + 对比对象 + 相对幅度**。

### 消融实验的写法

消融实验是审稿人最爱看的部分（证明你的每个模块都不是凑数的）。表格示例：

| 变体 | 模块 A | 模块 B | 模块 C | RMSE ↓ |
|---|:-:|:-:|:-:|---|
| Baseline | ✗ | ✗ | ✗ | 16.2 |
| + A | ✓ | ✗ | ✗ | 14.8 |
| + A + B | ✓ | ✓ | ✗ | 13.5 |
| Full | ✓ | ✓ | ✓ | **12.7** |

文字描述：必须**逐行解读**变化幅度，并把最大贡献模块拎出来。

## 7. 结论

### 结构

```
[1] 本文做了什么（用 1–2 句概述本文工作）
[2] 主要成果（按创新点顺序列举，每条对应一个具体结果）
[3] 应用价值（"为...提供..." 一句）
[4] 不足与展望（可选；很多顶刊鼓励写）
```

### 与摘要的差异

| 摘要 | 结论 |
|---|---|
| 给"还没读论文的人" | 给"刚读完论文的人" |
| 强调"我做了什么、有什么用" | 强调"我证明了什么、还能做什么" |
| 不引数据细节 | 可以引关键数据 |
| 不写未来工作 | 可以写未来工作 |

### 句式

- "本文针对 X 问题，提出了一种 Y 方法。"
- "通过 X 验证了 Y。"
- "数值算例表明 ..."
- "本文的工作为 X 提供了 Y 方面的支撑。"
- 展望："未来工作将关注 X，进一步探索 Y。"

## 8. 语言风格细则

### 工科论文动词库（按场景）

| 场景 | 动词 |
|---|---|
| 提出 | 提出、建立、构建、设计、发展、给出 |
| 改进 | 改进、扩展、增强、推广 |
| 比较 | 对比、对照、相较于 |
| 验证 | 验证、表明、证明、揭示、证实、说明 |
| 分析 | 分析、讨论、探究、考察、研究 |
| 测量 | 测量、采集、获取、记录 |
| 计算 | 计算、求解、推导 |
| 应用 | 应用于、适用于、用于 |
| 优化 | 优化、最小化、最大化 |

### 长句结构

工科论文允许长句，但要有"骨架可循"。最常见的两种：

**结构 1：分号递进**
> "首先 A，以达到 a；其次 B，构建 b；接着 C，实现 c。"

**结构 2：定语修饰主语**
> "[针对 X 问题的] [基于 Y 方法的] [Z 模型]，[在 W 条件下] 实现了 V 的优化。"

注意定语不要超过 3 层；超过 3 层就拆句。

### 必须规避的词组

| 分类 | 词 / 词组 | 修改方向 |
|---|---|---|
| 空洞过渡 | 综上所述、总而言之、值得注意的是、众所周知 | 删除，直接陈述 |
| 大词无实 | 全面提升、深度赋能、显著优化、智能化 | 用具体数字替换 |
| AI 套话 | 旨在、致力于、探索性地、一定程度上 | 删除 |
| 模糊量化 | 大量、许多、显著、明显 | 给具体数字或范围 |
| 过度修饰 | 广泛的、深入的、全面的、系统的 | 多数情况删除 |
| 口语 | 这样、那样、的话、就是、其实 | 改成书面语 |

### 中英文混排

- 英文术语首次出现给中文翻译 + 英文原文（缩写）：如"剩余使用寿命（remaining useful life, RUL）"
- 之后用 RUL 即可，不需要每次重复
- 中文标点和英文之间一般**不加空格**（GB/T 15834-2011），但有些期刊要求加，按目标期刊指南
- 英文人名大写：BENDSØE、JANG（顶刊常见的全大写姓氏样式）

### 数字的写法

- 表示数量、序号用阿拉伯数字：3 个、第 5 步、迭代 100 次
- 表示概数用汉字：三五个月、十几次
- 单位前用阿拉伯数字：5 mm、10 N、3.2 kg
- 整数数字单位前后空格按目标期刊指南

## 自查清单

写完任何一段后，对照以下清单：

- [ ] 这段回答了"问题、方法、创新、有效性"中的哪一个？答不上来 → 删/改
- [ ] 这段有没有可以量化的判断？有 → 必须给具体数字
- [ ] 段中所有形容词/副词去掉后，意思是否仍然成立？是 → 删
- [ ] 有没有连续 3 个被动句？有 → 改主动
- [ ] 有没有用本指南"必须规避的词组"？有 → 替换
