# 阶段 A：规划阶段 SOP

用户处于"想好做什么 / 还在选题 / 已选题但没动笔"阶段。本 SOP 帮助用户从模糊想法走到可执行的写作计划。

## 流程总览

```
1. 选题论证（这事值不值得做）
   ↓
2. 现状梳理（别人做到哪了）
   ↓
3. 创新点提炼（你比别人新在哪）  ←  此处可调用 references/innovation.md
   ↓
4. 技术路线（怎么做）
   ↓
5. 全文大纲（写什么）
   ↓
6. 投稿期刊预选
```

每一步都有可交付物。**不要跳步**——跳步是后期返工的最大原因。

---

## 步骤 1：选题论证

### 必须问用户的 5 个问题

1. **你这个题目要解决的是什么问题？** （不超过 30 字）
2. **这个问题为什么重要？** （工程价值或学术价值，至少一个）
3. **现有方法做得怎么样？** （满意 / 不满意；不满意在哪）
4. **你有什么资源？** （数据、设备、合作方、师兄/师姐积累）
5. **目标期刊层次？** （中文核心 / EI 中文 / 顶刊）

任何一项答不出 → 说明选题论证不充分，先停下来调研。

### 选题成熟度评级

根据上面 5 个问题的回答给出评级：

| 评级 | 特征 | 建议 |
|---|---|---|
| **成熟** | 5 项全清晰；问题具体；现有方法有明确不足 | 直接进入步骤 2 |
| **半成熟** | 3–4 项清晰；问题方向清楚但具体落点模糊 | 用"问题树"细化 |
| **不成熟** | < 3 项清晰；只有大方向 | 先回去做文献调研，1–2 周后再来 |

### "问题树"：从大方向到具体题目

如果用户说"我想做齿轮箱故障诊断"——这是大方向，不是题目。用问题树细化：

```
齿轮箱故障诊断（大方向）
├─ 故障类型
│   ├─ 齿面磨损
│   ├─ 齿根裂纹
│   └─ 轴承故障
├─ 数据源
│   ├─ 振动信号
│   ├─ 油液分析
│   └─ 多源融合
├─ 方法路线
│   ├─ 信号处理 + 特征工程
│   ├─ 传统机器学习
│   └─ 深度学习
├─ 工况
│   ├─ 稳态工况
│   ├─ 变转速 / 变载荷
│   └─ 启停过渡阶段（如李浩论文）
└─ 数据特性
    ├─ 标签充足
    ├─ 小样本
    └─ 跨工况迁移
```

引导用户在每个分支选一个具体路径，最后**组合出一个具体题目**。例：齿根裂纹 + 振动信号 + 深度学习 + 变转速 + 小样本 → "变转速工况下基于元学习的小样本齿根裂纹诊断方法"。

## 步骤 2：现状梳理

### 调研策略

**不要让用户瞎搜**。给一个调研框架：

1. **3 篇综述**：找近 3 年内目标方向的综述论文（中英文各 1–2 篇）
2. **5 篇代表作**：综述里高频引用的、或在目标期刊上的代表性论文
3. **3 个 GitHub / 数据集**：开源代码或公开数据
4. **2 个最新 SOTA**：最近半年内的顶会/顶刊新工作

调研要有产出：**一份现状梳理表**

```
| 工作 | 时间 | 期刊 | 核心贡献 | 优点 | 局限 | 与本文关系 |
|---|---|---|---|---|---|---|
| 张三 [N] | 2022 | 机械工程学报 | xxx | xxx | xxx | 本文将解决其...局限 |
```

### 调研深度的判断

调研到什么程度可以停？标准：
- 能列出该方向的**3 条主流技术路线**及各自代表作
- 能说清每条路线的**典型局限**
- 能找到**至少 2 个**"还没人做但应该做"的具体点

如果做不到，调研不充分。

### 调研工具建议

skill 不直接做文献检索（有专门工具），但可以建议用户用：
- 知网、万方（中文）
- IEEE Xplore、Web of Science、Scopus（英文）
- Google Scholar（前沿和被引）
- Connected Papers（可视化引用网络）

## 步骤 3：创新点提炼

→ 调用 `references/innovation.md` 中"二、创新点包装"部分（用户已经有想法）或"三、头脑风暴"（用户还没想法）。

**规划阶段的创新点要满足**：

1. **可写出 3 条**，每条独立成立
2. **3 条之间有逻辑关系**（递进或并列），不是 3 个零散的点
3. **每条都能映射到一个章节**（方法 1、方法 2、方法 3）
4. **总体落在 L2 以上层级**（参见 innovation.md 的层级定义）

### 创新点 vs 工作量

避免一个误区：**很多工作量 ≠ 创新性**。

- 做了 100 组实验 → 是工作量，不是创新点
- 处理了 10 GB 数据 → 是工作量，不是创新点
- 训练了 200 小时 → 是工作量，不是创新点

创新点只能是：**新方法 / 新理论 / 新发现**。把工作量当创新点写，审稿人一眼看穿。

---

## 步骤 4：技术路线

### 技术路线 vs 实验方案

- **技术路线**：从输入到输出的总体方法流向（一张图 + 一段话）
- **实验方案**：具体怎么验证（数据、对比、消融）

规划阶段先做技术路线，实验方案在写作阶段细化。

### 技术路线图的画法

要素：
- **输入**（数据 / 信号 / 参数）→ 用圆角矩形
- **预处理 / 特征工程**（如果有）→ 用矩形
- **核心方法模块**（你的创新点）→ 用粗边矩形或不同颜色
- **输出**（预测值 / 决策 / 优化结果）→ 用圆角矩形
- **评价指标**（侧边标注）

每个核心模块要标注**对应的创新点编号**：模块 A → 创新点 1，模块 B → 创新点 2 ...

### 技术路线图模板（伪代码）

```
[输入: 振动信号 x(t)]
        ↓
[1. 时频特征提取]   ← 创新点 1: 自适应小波包分解
        ↓
[2. 特征序列建模]   ← 创新点 2: 双向 LSTM + 注意力
        ↓
[3. 半监督预测]     ← 创新点 3: 变分自编码器
        ↓
[输出: 剩余寿命 + 不确定性]
```

提醒用户：**画在 PPT 上即可**，正式排版到论文里在写作阶段做。

## 步骤 5：全文大纲

### 大纲粒度

规划阶段的大纲应该到**三级标题**，每个三级标题下用 2–3 句话描述要写什么。

例：

```
1 引言
   1.1 研究背景与意义
       - 引出 X 在 Y 工程中的关键作用
       - 给出 1–2 个工程案例的代价/影响
   1.2 国内外研究现状
       1.2.1 方向 A 的相关工作
       1.2.2 方向 B 的相关工作
       1.2.3 当前研究的不足
   1.3 本文工作
       - 创新点 1, 2, 3 列举
       - 文章结构

2 问题描述与基础理论
   2.1 问题的数学描述
   2.2 关键基础概念

3 方法 - 创新点 1
   3.1 模块设计
   3.2 关键公式
   3.3 性质分析

4 方法 - 创新点 2 ... 

5 实验验证
   5.1 实验设置
       - 数据集
       - 评价指标
       - 对比方法
       - 实现细节
   5.2 主实验
   5.3 消融实验
   5.4 参数敏感性
   5.5 可视化分析

6 结论
   6.1 主要工作
   6.2 应用价值
   6.3 不足与展望
```

### 字数预估

中文核心论文常见字数 6000–10000 字（不含参考文献）。各部分占比：

- 引言：10–15%（800–1500 字）
- 问题描述：5–10%（500–1000 字）
- 方法：35–45%（每个创新点 1000–1500 字）
- 实验：25–35%（2000–3500 字）
- 结论：5%（300–500 字）

按比例反推每节字数，让用户写的时候心里有数。

## 步骤 6：投稿期刊预选

### 预选 3 本：第一志愿、保底、保险

- **第一志愿**：方向最匹配 / 影响因子最高 / 录用难度大
- **保底**：方向匹配 / 录用相对容易
- **保险**：方向勉强匹配 / 录用宽松（极端情况）

### 期刊选择因素

| 因素 | 重要性 | 如何判断 |
|---|---|---|
| 方向匹配 | 极高 | 翻最近 1 年目录看有无类似题目 |
| 影响因子/分区 | 高 | 中科院分区、JCR 分区、北大核心 |
| 录用周期 | 中 | 期刊官网公示或问导师/师兄 |
| 投稿费/版面费 | 视情况 | 部分期刊收费 |
| 是否本校认可 | 高（毕业用） | 本校学位认定文件 |
| 是否好引 | 中（影响后续被引） | OA 期刊更容易被引 |

### 中文工科常见顶刊（部分举例）

- **机械总刊**：机械工程学报、机械科学与技术、机械设计、振动工程学报
- **AI / 控制**：自动化学报、控制与决策、计算机学报、软件学报、模式识别与人工智能
- **测试 / 仪表**：仪器仪表学报、电子测量与仪器学报
- **能源 / 动力**：动力学报、动力工程学报
- **航空航天**：航空学报、宇航学报
- **机器人**：机器人

期刊选择最终决策权在用户。skill 不替用户决定。

---

## 规划阶段输出物

完成规划阶段后，应有以下产出（建议保存到一个文档）：

1. **题目草稿**（1 个主选 + 2 个备选）
2. **现状梳理表**（≥ 8 篇）
3. **创新点列表**（3 条，每条 What/Why/Effect）
4. **技术路线图**（PPT / Excalidraw）
5. **全文三级大纲**（每节有 2–3 句描述）
6. **目标期刊**（3 本 + 选择理由）
7. **预计完成时间表**（按步骤）

## 自查清单

规划阶段结束前对照：

- [ ] 题目是否符合 writing-patterns.md 中的命名公式？
- [ ] 创新点是否经过 innovation.md 中的"五个诊断问题"检验？
- [ ] 现状梳理是否覆盖了 ≥ 3 条主流技术路线？
- [ ] 技术路线图中每个核心模块是否对应一个创新点？
- [ ] 大纲的章节字数预估是否合理（无明显头重脚轻）？
- [ ] 目标期刊的最近 1 年目录里是否有相似主题（说明期刊确实接收这类）？
