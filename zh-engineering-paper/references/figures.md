# 中文工科顶刊插图规范

来源：5 篇代表性顶刊论文 50+ 张图的视觉规范提炼。

**核心原则**：图是为了**节省读者时间**，不是为了"显得做了很多工作"。一张好图能在 5 秒内让审稿人理解一个概念，差图会让审稿人怀疑你的整体严谨性。

## 整体规范

### 字体

| 元素 | 中文 | 英文/数字 | 字号（投稿原稿）|
|---|---|---|---|
| 图标题 | 宋体 | Times New Roman | 9–10.5 pt |
| 坐标轴标签 | 宋体 | Times New Roman | 9 pt |
| 刻度数字 | — | Times New Roman | 8–9 pt |
| 图例 | 宋体 | Times New Roman | 8–9 pt |
| 图内标注 | 宋体 | Times New Roman | 7–8 pt |
| 公式/符号 | — | Times New Roman 斜体 | 同上下文 |

**关键**：中英文不同字体；公式中变量斜体、数字和单位正体。

### 配色基调

中文工科顶刊主流是**克制配色**：

- **优先黑白线条**。能用黑白说清楚的不用彩色。
- **彩色不超过 5 种**。超过 5 种通常说明图太复杂，应拆图。
- **避免饱和度过高的纯红、纯绿、纯蓝**。用降饱和后的工程色。
- **色弱友好**。不用纯红 + 纯绿对比（约 8% 男性色弱看不出差异），改用红 + 蓝、橙 + 蓝、暖 + 冷。
- **打印兼容**。中文期刊很多读者打印阅读，单色打印时不同曲线/区域要可区分。

推荐基础色板（十六进制）：

```
主色：    #1F77B4 (深蓝)        #D62728 (砖红)         #2CA02C (草绿)
辅色：    #FF7F0E (橙)          #9467BD (紫)           #8C564B (棕)
中性：    #333333 (深灰)        #999999 (中灰)         #CCCCCC (浅灰)
背景：    #FFFFFF (白)          #F5F5F5 (浅灰底)       #E8E8E8 (表头底)
```

### 线宽

| 元素 | 线宽 |
|---|---|
| 主曲线 | 1.2–1.5 pt |
| 坐标轴 | 0.8 pt |
| 网格线 | 0.5 pt（虚线） |
| 流程图边框 | 1.0–1.2 pt |
| 工程图轮廓 | 0.8–1.0 pt |
| 工程图尺寸线 | 0.5 pt |
| 载荷/约束箭头 | 1.5–2.0 pt |

**全文线宽要协调**——如果一篇论文里有的图坐标轴很粗、有的很细，整体观感乱。

### 分辨率与格式

- **矢量优先**：PDF / EPS / SVG。中文期刊接受度由高到低：EPS > PDF > TIFF > PNG > JPG。
- **栅格化兜底**：PNG @ 600 DPI（线条图）、TIFF @ 300 DPI（照片）。
- **不要用 JPG 存线条图**——压缩瑕疵会把直线弄成"毛边"。
- 文件大小：单图通常 < 5 MB；过大投稿系统传不上去。

## 9 类图的具体规则

### A. 流程图 / 技术路线图

**用途**：方法整体框架、算法流程、研究路线

**规则**：
- 形状语义：**矩形 = 操作/模块**；**菱形 = 判断**；**圆角矩形 = 输入/输出**；**圆形 = 起止点**
- 主要箭头方向：**从上到下**或**从左到右**，全图一致
- 箭头：实线 + 标准三角形头，线宽 1.0 pt
- 颜色：黑边框白填充为基底；同类模块用同一种浅色填充（如同一阶段用浅蓝 #E8F4F8）
- 文字：宋体 9–10 pt，居中
- **关键约束**：避免折线交叉；交叉时一条用"小桥"跨过另一条
- **避免**：3D 风格阴影、渐变填充、过多颜色

**编号**：流程图各步骤可以编号（步骤 1、步骤 2），便于正文文字描述时引用。

### B. 网络结构图（神经网络）

**用途**：展示深度学习模型的架构

**规则**：
- 用堆叠矩形表示层；每层标注**层类型 + 维度**（H×W×C）
- 不同类型层用不同浅色背景区分：
  - 卷积/全连接：浅蓝 #D6EAF8
  - 注意力/Transformer：浅黄 #FCF3CF
  - 残差/跳连接：浅绿 #D5F5E3
  - 损失/输出：浅红 #FADBD8
- 数据流箭头：黑色或深灰，必须标方向
- 拼接 / 求和：用符号 `⊕`（拼接）或 `⊙`（按元素相乘）
- 注意力权重等"软量"：用小热力图嵌入
- **总宽 : 总高 ≈ 2:1 或 3:2**，太瘦或太方都不易读
- **关键约束**：尺寸标注全文统一格式（要么全用 H×W×C，要么全用 C×H×W）

**示例（朱显丞 CDHA 论文）**：网络主干 + 分支 + 多个注意力块，用 ⊕ 表示融合，每块下方标注通道数。

### C. 机械结构示意图

**用途**：展示零部件、装配关系、力学边界条件

**规则**：
- 视图：俯视图、侧视图、剖视图三选一；剖视图用 **45° 平行斜线** 填充
- 配色：
  - 零件本体：浅灰 #E0E0E0 或浅蓝 #D6E4F0
  - 油膜 / 流体：浅青 / 浅绿
  - 旋转构件：可加微弱径向渐变
- 载荷箭头：**红色 #D62728**，宽 2.0 pt，箭头比例 1:8
- 转速箭头：**绿色 #2CA02C**，弧线箭头
- 约束符号：标准固定支座（棋盘斜线）、滚动支座（圆形）、铰支座（三角形）
- 尺寸标注：尺寸线 0.5 pt，文字 8 pt 在尺寸线上方，单位标在数字后（mm、N）
- 坐标系：右下角 (x, y, z) 箭头，文字 8 pt
- **关键**：左图整体、右图局部放大；放大区域用红色细矩形框 1 pt 圈出

### D. 折线 / 性能曲线图

**用途**：迭代收敛曲线、时间序列、性能-参数关系

**规则**：
- **坐标轴**：四面闭合，刻度**朝内**（这是中文工科习惯，与 IEEE 朝外不同）
- 主刻度 + 次刻度都要有；网格只在主刻度，浅灰虚线 #CCCCCC
- 曲线线宽 1.2–1.5 pt
- **多曲线区分**：优先用**线型**（实线、虚线、点划线、点线）；颜色作为次要区分
  - 单色打印能区分是底线
- 数据点 marker：圆 ○、方 □、三角 △、菱 ◇；大小 5–6 pt；间隔放置（每 5–10 个数据点放一个 marker，避免遮蔽曲线）
- **图例**：右上或右侧；带边框（线宽 0.5–0.8 pt）；半透明白底；字号 8–9 pt
- 曲线条数 ≤ 4 条；超过 4 条要拆图或用子图
- 置信区间填充：alpha = 0.15–0.20，颜色与曲线同色但更浅

### E. 散点图 / 分布图

**用途**：模型对比（性能 vs 参数量）、特征空间可视化、相关性

**规则**：
- 标记：圆形 ●，统一大小或按"第三维"变化（参数量、模型规模）
- 单类：黑色 #333333，alpha 0.6–0.7
- 多类：用前述基础色板，每类一种颜色 + 一种 marker 形状（双重区分，色弱也能读）
- 边框：标记内填色 + 细黑边框 0.5 pt
- 趋势线（可选）：虚线 #999999，1 pt，标注拟合公式与 R²
- **避免**：标记重叠严重时还硬画散点；改用密度图或 hex bin

### F. 柱状图 / 对比图

**用途**：消融实验结果、方法对比

**规则**：
- 柱宽 / 柱间距 ≈ 1 : 0.3
- 单组：统一颜色（深蓝 #1F77B4）
- 多组（如多数据集 × 多方法）：分组并列；同一方法跨组用同色
- 柱顶标数值，字号 8 pt，距柱顶 2–3 pt
- Y 轴**通常**从 0 开始；如果数据差异小（如 0.95 vs 0.98），可以从 0.9 开始但要在 Y 轴上画**截断符号**（//）
- 误差棒：黑色 1 pt，帽宽 = 柱宽 × 0.3
- X 轴标签长度大就**旋转 30°–45°**（不要垂直旋转，难读）

### G. 云图 / 热力图（应力、压力、温度分布）

**用途**：场量分布、注意力权重

**规则**：
- 配色：
  - **物理场量（应力/压力/温度）**：用 viridis 或 RdYlBu_r（蓝→红）。**避免 jet（彩虹色）**——它在科学绘图中已被诟病，色弱不友好且感知不均匀
  - **注意力权重 / 概率**：用单调的 Blues / Greens / Reds
- Colorbar：右侧；高度与图齐；宽度 = 图宽 × 5–8%；间距 3–5 px
- Colorbar 标签：上方或右侧标量名 + 单位（如 "压力 / MPa"），字号 9 pt
- Colorbar 刻度：均匀分布，5–7 个刻度值
- 等值线（contour lines）：黑色细线 0.5 pt，标注关键数值
- 坐标轴显示空间坐标（mm / m），单位标在轴标签
- **避免**：用 jet 配色、colorbar 太宽、刻度过密

### H. 图像对比（超分、重建、分割）

**用途**：算法效果的视觉对比

**规则**：
- 排版：**网格状并列**，每行同一样本，每列同一方法
- 第一列**真值（GT）或原始**，最后一列**本文方法**
- 子图大小**严格相同**
- 子图间距 5–10 px
- 子图下方标方法名，宋体 8–9 pt
- 关键指标（PSNR、SSIM）：每个子图右下角用半透明白底标注，字号 7–8 pt
- 局部放大：用红色矩形框（1.5–2 pt）标定区域，放大版本作为**插入图**置于角落或单独成行
- **避免**：JPG 格式（会引入压缩瑕疵）；子图间不对齐

### I. 表格

虽然不是图，但版式同样重要：

**规则**：
- **三线表**：上、中、下三条横线，无竖线（中文工科顶刊标准）
  - 顶线 1.0 pt
  - 中线（表头与数据分隔）0.75 pt
  - 底线 1.0 pt
- 表头：浅灰背景 #E8E8E8，加粗
- 数值列：右对齐，小数位数全表统一
- 文本列：左对齐
- 单位放在表头："参数 / 单位"格式，如 "刚度 / (N·m⁻¹)"
- **本文方法**所在行：加粗或浅黄底色 #FFF9E6 区分
- 最优值：加粗 + 下划线
- 次优值：仅加粗
- 表格下方可加注解：用 **Note** 或 **注：** 起头，字号 8 pt

## matplotlib 配置

skills 包含可直接 import 的脚本：`scripts/paper_plot.py`

```python
import sys
sys.path.append(r"C:\Users\Full stop\.agent\skills\zh-engineering-paper\scripts")
from paper_plot import setup_paper_style, COLORS, save_paper_figure

setup_paper_style()  # 一行配置好顶刊风格

fig, ax = plt.subplots(figsize=(8, 6))
ax.plot(x, y1, label='方法 A', linestyle='-', color=COLORS['primary'])

```

### 直接写 matplotlib 时的关键设置

如果不用模板而是手写，至少保证以下几条：

```python
import matplotlib.pyplot as plt
from matplotlib import rcParams

rcParams['font.sans-serif'] = ['SimSun', 'Times New Roman']  # 中文宋体
rcParams['font.family'] = 'serif'
rcParams['axes.unicode_minus'] = False  # 负号正常显示
rcParams['figure.dpi'] = 300
rcParams['savefig.dpi'] = 600  # 保存高 DPI
rcParams['savefig.bbox'] = 'tight'

# 字号
rcParams['font.size'] = 9
rcParams['axes.labelsize'] = 9
rcParams['xtick.labelsize'] = 8
rcParams['ytick.labelsize'] = 8
rcParams['legend.fontsize'] = 8

# 坐标轴
rcParams['axes.linewidth'] = 0.8
rcParams['xtick.direction'] = 'in'   # 关键：朝内
rcParams['ytick.direction'] = 'in'
rcParams['xtick.minor.visible'] = True
rcParams['ytick.minor.visible'] = True
rcParams['xtick.top'] = True   # 上下左右都有刻度
rcParams['ytick.right'] = True

# 网格
rcParams['axes.grid'] = True
rcParams['grid.linestyle'] = '--'
rcParams['grid.linewidth'] = 0.5
rcParams['grid.color'] = '#CCCCCC'
rcParams['grid.alpha'] = 0.7

# 线宽
rcParams['lines.linewidth'] = 1.5

# 图例
rcParams['legend.frameon'] = True
rcParams['legend.edgecolor'] = '#666666'
rcParams['legend.framealpha'] = 0.9
```

### 中文字体可能的坑

Windows 上 SimSun 通常正常工作。Linux 上需要安装：

```bash
# Ubuntu
sudo apt install fonts-wqy-zenhei
# 或下载 SimSun 字体
```

如果遇到中文显示问号 / 方框，检查：
1. matplotlib 字体缓存：`import matplotlib; matplotlib.font_manager._rebuild()`
2. 系统是否安装了 SimSun
3. 改用 'STSong' 或 'Microsoft YaHei' 作为备用

## 图前/图后的文字描述

### 图标题（Figure Caption）

**结构**：图号 + 简洁描述 + (可选) 子图说明

✅ 标准：**图 5  优化收敛曲线**
✅ 带子图：**图 7  风电齿轮箱启动阶段油膜压力分布。(a) t = 0.5 s；(b) t = 1.0 s；(c) t = 2.0 s**
❌ 太长：图 5 展示了我们方法在 NASA 数据集上的迭代过程中目标函数值的变化情况

**位置**：**图下方**（中文工科论文一律图下、表上）

### 正文中如何引用图

每张图必须在正文中**至少引用一次**。引用句式：

- "如图 3 所示，..."
- "图 5 给出了 ... 的对比结果。从图中可以看出 ..."
- "结合图 7（a）和图 7（b），..."

引用时要**解读**，不要只说"如图 X 所示"——要点出读者应该从图里看到什么。

### 图后的解读段落

每张关键图后面应该有一段 50–150 字的文字解读，结构：

```
[现象描述] 图 X 表明...
[定量提取] 在 ... 条件下，... 达到 ... ；相比 ... 减少了 ...%
[机理解释] 这是由于 ...
[与现有方法对比] 相较于 ...，本方法 ...
```

## 常见错误

| 错误 | 修正 |
|---|---|
| 用 jet（彩虹色）做应力云图 | 改 viridis 或 RdYlBu_r |
| 折线图 6 条曲线挤一起 | 拆成 2 张子图，每张 3 条 |
| 散点图标记 > 200 个全部画出 | 用密度图或随机抽样可视化 |
| 图例挡住数据 | 移到图外或选数据稀疏区 |
| 中文字符显示成方框 | matplotlib 字体配置 |
| 图分辨率太低导致打印模糊 | savefig dpi 提到 600 或导出矢量 |
| 子图大小不一 | 统一 figsize；用 `gridspec` 控制 |
| 坐标轴单位缺失 | 任何带量纲的轴都标 "量名 / 单位" |
| 颜色不区分（都偏蓝） | 检查色板，用差异度高的色 |
| 表格用十字格 | 改三线表 |
| 图没有在正文引用 | 每张图至少在正文出现一次 |

## 自查清单

提交图前对照：

- [ ] 字体：中文宋体 / 英文 Times，全文统一
- [ ] 字号：标题 ≥ 标签 ≥ 刻度，符合层级
- [ ] 配色：≤ 5 种主色；色弱友好；单色打印可区分
- [ ] 线宽：曲线 1.2–1.5 pt；坐标轴 0.8 pt；网格 0.5 pt
- [ ] 坐标轴：四面闭合；刻度朝内；带次刻度
- [ ] 单位：所有量带轴的都有"量名 / 单位"
- [ ] 图例：位置不挡数据；带边框；字号合适
- [ ] 分辨率：矢量优先；栅格 600 DPI
- [ ] 图标题：在图下方；编号正确；不冗长
- [ ] 正文引用：每张图至少出现一次；有解读
