# 本地月报模板（Markdown）

```markdown
---
tags:
aliases:
---
# {{M月-YY}}

## 本月主线与阶段推进

### {{按主线动态命名的小节}}
{{先用 1 段话概述本月主线，再写上中下旬或 W1->W4 的阶段变化。}}
- {{阶段推进点 1}}
- {{阶段推进点 2}}
- {{阶段推进点 3}}

### {{如存在第二主线，再增加小节}}
{{继续用“概述 + 适量要点”的方式展开。}}
- {{阶段推进点 1}}

## 关键里程碑与量化结果
- {{里程碑 1：可验证产出 + 指标}}
- {{里程碑 2：版本/提交/实验/上线结果 + 价值}}

## 关键决策与策略调整
- {{决策 1：触发原因 -> 调整动作 -> 当前结果}}
- {{决策 2：方案取舍 -> 风险权衡 -> 后续验证}}

## 问题与风险（含处理状态）
- {{问题 1：现状 + 影响 + 状态（已解决/进行中/待决策）}}
- {{问题 2：现状 + 影响 + 状态（已解决/进行中/待决策）}}

## 下月目标与执行计划
- {{P1 目标：动作 + 完成标准}}
- {{P2 目标：动作 + 完成标准}}
- {{P3 目标：动作 + 完成标准}}
```

## 使用提醒

- 文件名使用 ISO 月份：`YYYY-MM.md`
- 正文标题使用旧标题格式：`# M月-YY`
- 第 1 章必须体现阶段变化，不能只平铺任务清单
- 第 2 章优先写可核验锚点，例如版本号、批次名、数量指标、提交记录
- 第 4 章的每条风险都要写处理状态
